# 유닛 테스트 보고서: 저장된 설정·읽기 위치·북마크

**날짜:** 2026-05-11
**범위:** `js/app/storage.js` — 검색 이력 외 저장 영역 전부
**관련 PR:** [#106](https://github.com/anglican-kr/common-bible/pull/106)

## 한 줄 요약

앱이 휴대폰·브라우저 안에 보관하는 거의 모든 정보(읽던 위치, 듣던 오디오 시간, 글자 크기, 색, 화면 모드, 책 정렬 순서, 시작 화면, 북마크, 설치 안내 표시 횟수 등)에 대해, 잘못된 값이 들어와 있거나 저장에 실패해도 앱이 멈추지 않고 안전한 기본값으로 복구되는지 자동으로 검증하기 시작했다.

## 이 변경으로 무엇이 더 안전해졌는가

자동 검증 케이스 수가 **245건 → 309건**으로 늘었고, 저장 영역 단독으로는 **19건 → 83건**.

이번에 자동으로 점검하게 된 동작:

### 읽던 위치 / 오디오 시간

- 책을 읽다가 앱을 닫고 다시 열면 같은 곳을 다시 보여주는지
- 오디오를 듣다가 멈췄을 때, 같은 책·같은 장이면 그 시점부터, 다른 책으로 가면 처음부터 시작하는지
- 저장 자체가 실패해도(예: 사파리 무한 새 탭 모드 같은 환경) 앱이 멈추지 않는지

### 외관 설정 — 글자 크기·색·다크모드

- 글자 크기에 16/18/20/22/24가 아닌 엉뚱한 값이 들어 있어도 → **18**(기본)로 안전하게 시작
- 색 이름이 모르는 값(예: "에메랄드")이면 → **네이비**로 폴백
- 다크/라이트/시스템 외 알 수 없는 값이 있으면 → **시스템 설정 따라가기**로 폴백

### 책 정렬 / 시작 화면

- 책 정렬 순서가 정상값("canonical" / "vulgate") 외 이상한 값이면 → **canonical**(기본)로 폴백
- 시작 화면 설정이 비어 있으면 → **마지막 위치 이어 읽기**가 기본

### 북마크 저장·불러오기

- 다른 기기에서 동기화된 북마크가 있으면 그쪽을 먼저 사용
- 동기화가 꺼져 있으면 이 기기에 저장된 것을 사용
- 저장 파일이 깨져 있어도 빈 목록으로 시작 (앱이 죽지 않음)
- 북마크를 저장할 때마다 **다른 기기 동기화 신호**가 정확히 한 번씩만 발사되는지

### 설치 안내 / 영구 저장 요청

- iOS 사용자에게 "홈 화면에 추가" 안내를 몇 번 보여줬는지 기억하는 부분
- 앱을 처음 본격적으로 쓰기 시작할 때 브라우저에 "이 앱 데이터는 함부로 지우지 말아 줘" 요청을 정확히 **딱 한 번만** 보내는지(실수로 매번 보내면 사파리가 사용자에게 매번 알림을 띄움)

## 발견된 이슈

없음. 위 모든 동작이 의도한 대로 동작함을 확인.

## 다음 의제

저장 부분은 마무리됐고, 이번에 손대지 않은 부분은 다음 단계로 미뤄둠:

- **공통 도구 모음** (창 닫기·포커스 잡기 등)
- **설정 메뉴 화면** (글자 크기·색 고르기 UI 자체)
- **설치 안내 모달**
- **책 목록·장 선택·헤더** 화면 일부
- **북마크·검색 화면**의 UI 부분

이 영역들은 별도 PR로 단계별로 진행할 예정.
