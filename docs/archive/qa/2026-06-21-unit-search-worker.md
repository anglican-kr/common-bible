# 유닛 테스트 보고서: 검색 엔진 핵심 로직 (검색 워커)

**날짜:** 2026-06-21
**범위:** `js/search-worker.js` — 검색을 백그라운드에서 처리하는 워커의 순수 로직: 검색어 해석(`in:책` 연산자), 성경 구절 주소 인식, 본문 부분일치, 결과 페이지 나누기
**관련 PR:** (작성 중)

## 한 줄 요약

검색은 화면이 멈추지 않도록 별도 작업 스레드(워커)에서 돈다. 그 워커의 핵심 4가지 계산 — 검색어에서 "특정 책 안에서만" 조건을 떼어내기, "창세 1:3" 같은 구절 주소 알아채기, 본문에서 단어 찾기, 결과를 쪽 단위로 나누기 — 이 지금까지 자동 검증 대상이 아니었는데, 이번에 전부 자동으로 점검하기 시작했다.

## 이 변경으로 무엇이 더 안전해졌는가

자동 검증 케이스 수가 **758건 → 781건**으로 늘었다 (+23, 회귀 0).

### 검색어 해석 (`in:` 연산자)
- 평범한 검색어는 그대로 통과
- `사랑 in:요한` → "요한"을 책 범위로 떼어내고 검색어는 "사랑"만 남김
- `in:` 뒤 공백 허용 (`in: 요한`도 동일)
- `in:`이 앞에 와도 동작 (`in:gen 사랑`)
- 여러 `in:`은 "또는"으로 누적 (여러 책 동시 범위)
- 영문 책 약칭은 대소문자 무시 (`in:GEN` = `in:gen`)
- 모르는 책 이름은 "해석 못 함"으로 따로 표시하고 범위는 비움
- 검색어 중복 공백은 하나로 정리

### 구절 주소 인식
- `창세 1:3` → 창세기 1장 3절로 정확히 해석
- `창세 1:3-11` → 절 범위 인식
- 전체 한글 책 이름(`요한복음 3:16`)으로도 매칭
- 영문 id 대소문자 무시 (`Gen 1:1`)
- 구절 주소 형식이 아니면 그냥 일반 검색어로 취급 (null)
- 존재하지 않는 책 이름이면 주소로 보지 않음

### 본문 찾기 (부분일치 + 책 범위 필터)
- 검색어가 절 본문에 들어 있으면 매칭 (대소문자 무시)
- "이 책들 안에서만" 범위가 있으면 그 책들만 검사
- 매칭이 없으면 빈 결과, 아직 안 불러온 데이터 묶음은 건너뜀
- 매칭된 각 절에 책·장·절·본문이 정확히 담김

### 결과 쪽 나누기
- 첫 쪽은 정해진 개수만큼, 전체 건수는 따로 보존
- 둘째·마지막 쪽이 올바른 구간을 가져옴 (마지막 쪽은 남은 만큼만)
- 범위를 벗어난 쪽 번호는 빈 결과지만 전체 건수는 그대로

## 발견된 이슈

없음. 위 모든 로직이 의도한 대로 동작함을 확인. 추가로 실제 앱에서 "사랑"을 검색해 워커 전체 파이프라인이 정상 동작함을 확인했다 (총 709건 / 102쪽으로 집계되고 실제 절 결과가 렌더됨, 오류 0건).
