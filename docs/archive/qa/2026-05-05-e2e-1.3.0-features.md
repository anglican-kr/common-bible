## 3. 신규 시나리오

### 3-1. `tests/e2e/test_bookmark_swipe.py` (6건)

| 테스트 | 검증 내용 |
|--------|---------|
| `test_swipe_reveals_mobile_actions` | 모바일 390px viewport에서 북마크 행 좌스와이프 → `.bm-swiped` 클래스 + 수정/삭제 버튼 노출 |
| `test_swipe_other_row_closes_previous` | 두 번째 행 스와이프 시 첫 번째 행 자동 닫힘 |
| `test_longpress_reveals_mobile_actions` | 600ms 롱프레스 → 스와이프와 동일하게 액션 패널 열림 |
| `test_short_press_does_not_reveal_actions` | 200ms 짧은 press → 액션 미노출 |
| `test_swipe_no_effect_on_desktop` | 1280px 데스크톱 viewport → canSwipe=false → 스와이프 무효 |
| `test_delete_via_swipe` | 스와이프 → 삭제 버튼 → confirm 승인 → syncStoreV2에서 북마크 제거 |

**구현 노트:** Playwright `page.mouse` API는 `setPointerCapture`와 함께 동작이 불안정하여 PointerEvent를 JS에서 직접 `dispatchEvent`로 주입하는 방식 채택.
