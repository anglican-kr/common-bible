# 유닛 테스트 보고서: 주소(URL) 해석 — 어떤 화면을 열지 결정하는 로직

**날짜:** 2026-06-21
**범위:** `js/app/routing.js` — 브라우저 주소창의 경로(`/창세기/3/16` 같은)를 읽어 "어떤 화면을, 어떤 상태로 열지"를 결정하는 `parsePath` 함수
**관련 PR:** (작성 중)

## 한 줄 요약

이 앱은 한 페이지짜리(SPA) 구조라, 주소창의 경로만 보고 성경 책 목록·장 화면·검색·북마크·설정 중 무엇을 띄울지, 또 어느 절을 강조 표시할지를 코드가 직접 해석한다. 그 해석을 담당하는 핵심 함수 `parsePath`가 지금까지 자동 검증 대상이 아니었는데, 이번에 전 갈래를 자동으로 점검하기 시작했다.

## 이 변경으로 무엇이 더 안전해졌는가

자동 검증 케이스 수가 **734건 → 758건**으로 늘었다 (+24, 회귀 0).

`parsePath`는 주소 한 줄을 받아 화면 종류와 부가 정보를 돌려준다. 이번에 자동으로 점검하게 된 갈래:

### 기본·탭 목적지
- 루트(`/`)와 빈 경로는 성경 책 목록 화면
- `/bookmarks` → 북마크 화면, `/settings` → 설정 화면

### 검색 (`/search`)
- 검색어·페이지 번호·"이 책들 안에서만"(in) 범위를 주소에서 정확히 꺼냄 — 그래서 검색 결과 링크를 공유하거나 뒤로가기 해도 같은 결과가 재현됨
- 페이지 번호가 깨진 값이거나 0이면 안전하게 1쪽으로 폴백
- 범위(in)에 빈 값이 섞여 들어와도 걸러냄

### 북마크 모아 읽기 (`/read`)
- `/read` → 전체 북마크 이어 읽기
- `/read/<폴더>` → 해당 폴더만 이어 읽기 (폴더 이름의 특수문자 디코딩 포함)

### 책 vs 분류 구분
- 알려진 분류 id(구약·외경·신약)는 분류 화면으로
- 그 외 한 단어 경로는 책 장 목록 화면으로 (분류와 책 이름을 헷갈리지 않음)

### 장 화면 + 절 바로가기 (딥링크)
- `/<책>/prologue` → 머리말 화면
- `/<책>/<장>` → 장 화면 (절 강조 없음)
- `/<책>/<장>/<절>` → 해당 절 강조, `16-20` 같은 범위 강조
- 역순으로 적어도(`20-16`) 자동으로 16~20으로 바로잡음
- `16-16`처럼 같은 절 범위는 단일 절로 축약
- `10-12,1-3`처럼 흩어진 여러 구간도 정렬·정규화해 `1-3,10-12`로 정돈 (실제 절 해석 로직을 그대로 불러 검증)
- `3a,3b` 같은 절 일부 표기도 보존
- `?hl=...`(검색어 강조)·`?resume`(이어 읽기) 같은 부가 옵션도 정확히 전달

## 발견된 이슈

없음. 위 모든 갈래가 의도한 대로 동작함을 확인. 추가로 실제 브라우저에서 주요 경로(장·검색·북마크·설정·절 범위)를 직접 띄워 정상 동작과 오류 0건을 확인했다.

## 다음 의제

`routing.js`의 나머지(`navigate`·`route`·페이지 메타/통계·스크롤 위치 추적)는 브라우저 히스토리·DOM에 묶여 있어 유닛보다 e2e가 맞는 경계다. 다른 모듈의 순수 로직 미커버(예: 오디오 시간 표시 `formatTime`)는 후속 소규모 추가 후보.
