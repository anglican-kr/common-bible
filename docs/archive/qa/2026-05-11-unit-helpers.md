## 한 줄 요약

앱이 모든 화면에서 공통으로 쓰는 작은 도구들 — 화면 요소 만들기·비우기, 모달이 열렸을 때 뒤쪽 화면을 키보드·낭독기로부터 격리하기, 모달 안에서 Tab 키가 바깥으로 새지 않게 가두기 — 이 의도대로 동작하는지 자동으로 확인하기 시작했다.

## 이 변경으로 무엇이 더 안전해졌는가

자동 검증 케이스 수가 **309건 → 340건**으로 늘었고, 공통 도구 단독으로는 **0 → 31건**.

이번에 자동으로 점검하게 된 동작:

### 화면 요소 만들기 (`el`)

- 태그 이름·속성·자식을 한 번에 받아 화면 요소를 깔끔히 만드는 함수
- 자식 자리에 빈 값(null/undefined)이 들어와도 그냥 건너뛰고 정상 작동
- 글자(텍스트), 다른 요소 노드, 또 다른 자식 요소가 섞여 있어도 순서대로 잘 붙음
- 숫자나 true/false가 속성 값으로 들어와도 문자로 안전하게 변환

### 화면 비우기 (`clearNode`)

- 어떤 영역의 자식 요소를 한 번에 제거 (예: 검색 결과 목록을 새로 그리기 전에 비우기)
- 비어 있는 영역에 호출해도 오류 없이 그냥 지나감
- 자식의 자식까지 강제로 끊지 않고 한 단계만 정리

### 모달 열렸을 때 뒤쪽 화면 격리 (`setInert`)

- 모달이 열리면: 뒤쪽 메인·사이드바·내비게이션을 키보드 Tab 이동 대상에서 빼고, 화면 낭독기에게도 "여긴 안 보이는 영역"이라고 알림
- 모달이 닫히면: 위 두 가지를 깔끔히 원복
- 한 번에 여러 종류의 영역을 선택해 동시에 격리/해제

### Tab 키 가두기 (`trapFocus`)

모달이 열렸을 때 Tab 키를 누르며 돌아다녀도 모달 바깥으로 빠져나가지 않게 잡아두는 기능. 접근성에 매우 중요한 부분.

- 마지막 버튼에서 Tab → 첫 번째로 돌아옴
- 첫 번째에서 Shift+Tab → 마지막으로 돌아옴
- 모달이 막 열렸을 때(아직 어떤 버튼에도 들어가지 않은 상태)에서 Shift+Tab을 눌러도 마지막 요소로 안전하게 진입
- 중간 버튼에서 Tab을 누르는 평범한 경우는 브라우저 기본 동작에 맡기고 간섭하지 않음
- 비활성화된 버튼(`disabled`)이나 키보드 이동 대상에서 빠진 요소(`tabindex=-1`)는 건너뜀
- 모달이 닫히면 Tab 가두기를 깨끗이 해제 (이후에는 일반 동작)

### 그 밖에 (`_$`, `chUnit`)

- `_$`: id로 화면 요소 찾기 — 없으면 안전하게 null 반환
- `chUnit`: 시편이면 "편", 그 외 책은 "장" — 작지만 화면 곳곳에서 쓰이는 단위 표시 함수
