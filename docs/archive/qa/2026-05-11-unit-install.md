# 유닛 테스트 보고서: 앱 설치 안내

**날짜:** 2026-05-11
**범위:** `js/app/install.js` — 사용자가 어떤 기기·브라우저에서 접속했는지 알아내고, 적절한 시점에 "홈 화면에 추가" 안내를 띄울지 결정하는 부분
**관련 PR:** (작성 중)

## 한 줄 요약

사용자가 어느 환경(아이폰 사파리, 안드로이드 크롬, 데스크톱, 미지원 브라우저 등)에서 접속했는지 정확히 알아내고, 안내 모달을 띄울 만한 시점인지 자동으로 판단하는 로직 — 이 의도대로 동작하는지 자동으로 확인하기 시작했다.

## 이 변경으로 무엇이 더 안전해졌는가

자동 검증 케이스 수가 **366건 → 406건**으로 늘었다 (+40).

이번에 자동으로 점검하게 된 동작:

### 환경 감지

사용자의 브라우저·기기를 정확히 분류:

- **이미 앱으로 설치된 상태**: 이미 홈 화면에서 실행 중이면 안내 자체를 안 띄움 (3가지 신호 모두 인식: 표준 표시 모드, 전체 화면 표시 모드, iOS의 navigator.standalone)
- **iPhone/iPad Safari**: "공유 → 홈 화면에 추가" 안내가 적합한 환경
- **iPad에서 데스크톱 모드 켜진 경우**: 사파리가 데스크톱처럼 보이지만 터치 신호로 정확히 iPad임을 감지
- **iOS의 다른 브라우저 (Chrome/Firefox/Edge/Opera/Google 앱)**: 사파리에서만 설치 가능하므로 "Safari로 열어주세요" 안내
- **안드로이드**: 브라우저 자체 설치 프롬프트 사용 가능
- **데스크톱 (Chrome/Edge)**: 주소창 옆 설치 아이콘 안내. Edg(새 Edge)는 지원, Edge/(레거시 Edge)는 미지원으로 정확히 구분
- **데스크톱 Firefox·Safari·Mac 사파리**: 미지원으로 안내 자체 회피

### 설치 가능 신호 추적

브라우저가 "지금 설치 가능합니다"라고 알려주는 순간 정확히 포착:

- 브라우저가 알려주면 즉시 보관해뒀다가, 사용자가 누르면 그 시점에 정확히 시스템 설치창을 띄움
- 사용자가 사라진 안내(다시 안 보겠다고 한 경우 등)에 대해서는 제대로 정리 — 같은 안내가 두 번 안 뜨도록 한 번 쓴 신호는 폐기
- 사용자가 실제로 설치를 완료하면 그 사실을 즉시 반영해 안내 상태를 갱신

### 안내 모달 자동 노출 (Nudge)

사용자가 거슬리지 않을 만큼만 안내가 뜨도록 정교하게 조절:

- 안내가 의미 있는 환경(iPhone Safari, 안드로이드)에서만 트리거. 데스크톱·미지원·이미 설치됨에서는 절대 안 뜸
- 사용자가 "다시 보지 않기"를 선택했으면 그 후로는 절대 안 뜸 (방문 횟수도 안 올라감)
- 처음에는 1회 방문에 보여주고, 닫으면 3회 뒤에 다시 — 점진적으로 간격이 벌어짐
- 페이지 로드 직후 화면을 가리지 않도록 1.5초 뒤에 부드럽게 등장
- 이 1.5초 사이에 사용자가 실제로 설치를 마쳤다면, 안내는 뜨지 않음
- 같은 페이지를 빠르게 새로 고침해도 중복으로 안 뜸 (저장된 카운터를 매번 다시 읽어 정확한 다음 시점을 계산)

### 구독자 관리 (개발자용 후크)

다른 화면(예: 설정 페이지의 "설치하기" 버튼)이 현재 설치 상태를 실시간으로 따라갈 수 있도록 하는 알림 시스템:

- 구독한 콜백이 예외를 던져도 다른 구독자에게 영향이 안 가도록 격리
- 구독 해제 후에는 더 이상 호출 안 됨
- 표시 모드 변경(예: PWA로 전환된 순간)도 즉시 알림으로 전파

## 발견된 이슈

없음. 위 모든 동작이 의도한 대로 동작함을 확인.

## 다음 의제

이번 PR로 install.js의 **로직 부분(환경 감지 + Nudge)은 모두 자동 점검 대상**. 다음 의제들은 아직 미커버:

- **모달 열기/닫기 자체** (`openInstallModal` / `closeInstallModal`) — 화면 잠금, 포커스 이동, 배경 격리 등 DOM 영역. e2e 테스트 또는 jsdom 도입 시 검토
- **모달 본문의 슬라이더** (iOS 사파리 안내의 3단계 사진 슬라이더) — 터치·스와이프·자동 진행 — jsdom 또는 e2e
- **북마크 화면**의 UI 영역 — 다음 PR에서 다룰 예정

## 부록: 개발자용 세부

| 항목 | 값 |
| --- | --- |
| 테스트 러너 | Node 24 자체 `node --test` |
| 격리 | `node:vm` per-test 컨텍스트, ADR-013 |
| 명령어 | `node --test tests/unit/install.test.js` |
| 단독 실행 | 40 통과 / 87ms |
| 전체 회귀 | 406 통과 / 31s |

신규 BEGIN/END 마커 영역 (`js/app/install.js`):
- `INSTALL_STATE` — 플랫폼 감지 IIFE + beforeinstallprompt/appinstalled/display-mode 리스너
- `NUDGE` — `maybeShowInstallNudge` 자동 노출 결정 로직

테스트 패턴: `window.matchMedia` + `navigator.userAgent`/`platform`/`maxTouchPoints` 스텁. `setTimeout`은 큐로 가로채서 결정적 발화. `const install = (...)` IIFE는 vm globalThis에 자동 노출되지 않으므로 슬라이스 끝에 `globalThis.install = install` 한 줄 추가.

DOM 의존이 적은 영역에 집중 — 모달 안의 슬라이더, 스크롤 잠금, 포커스 복귀 등 DOM-anchor 헤비한 부분은 의도적으로 제외(ADR-013 dual-track 기조). buildInstallBody의 platform별 렌더 스위치는 UI 변경에 취약해 영구 스킵.
