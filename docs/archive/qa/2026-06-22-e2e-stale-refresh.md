# e2e 보고서: 브라우저 자동 테스트 사전 실패 전수 진단·해소

**날짜:** 2026-06-22
**범위:** `tests/e2e/` 전체 — 그동안 "환경 탓"으로 미뤄둔 사전 실패 묶음 진단 + 코드와 어긋난 낡은 테스트 정리
**관련 PR:** (작성 중)

## 한 줄 요약

브라우저를 실제로 띄워 동작을 확인하는 자동 테스트(e2e)에서 한동안 실패하던 묶음들을 끝까지 파봤다. 결과: 의뢰받은 3묶음은 **낡은 브라우저 버전 탓**이라 브라우저를 새로 깔자 저절로 통과했고, 그 과정에서 드러난 다른 12건은 **앱이 의도대로 바뀌었는데 테스트가 옛 동작을 붙들고 있던 것**(앱 버그 아님)이라 현행 동작에 맞게 고쳤다. 이제 전체 e2e가 **215개 전부 통과**한다.

## 무엇을 확인했나

### 1. 의뢰받은 3묶음 — 낡은 브라우저 버전 문제 (앱 정상)

탭 바 검색 모핑(`test_tabbar.py`), 설정 팝오버·캐시 비우기(`test_settings.py`), 북마크 폴더 펼치기(`test_bookmark_folders.py`) — 이 세 묶음은 예전 headless 브라우저에서 클릭·전환이 제대로 안 일어나 실패했다. 최신 브라우저(Chrome Headless Shell 148)를 설치하자 **전부 통과**: 탭바 11/11, 설정 3/3, 폴더 2/2. 앱 코드 문제가 아니었다.

### 2. 전체를 돌리니 드러난 낡은 테스트 12건 (앱은 의도대로, 테스트가 옛것)

브라우저를 새로 깐 김에 전체를 돌렸더니 다른 12건이 실패했다. 하나하나 실제 동작을 확인한 결과 **전부 앱이 의도적으로 바뀐 부분을 테스트가 못 따라간 것**이었다:

- **책 이름 짧게 표시 4건** — 이제 복음서(마태오/마르코/루가/요한)도 좁은 화면에서 짧은 이름으로 바뀌는데, 테스트는 "복음서는 정식 이름 그대로"라는 옛 규칙을 검사하고 있었다. 또 터치 기기에서는 다른 방식(미디어쿼리)으로 짧은 이름을 보여주고, 책 목록 격자가 글자 크기에 따라 칸을 넓혀 예전 조건으로는 더 이상 줄바꿈이 안 일어났다. 현행 동작에 맞게 고치고, "복음서도 짧게 바뀐다"는 새 확인을 추가했다.
- **길게 눌러 절 선택 1건** — 절을 길게 눌러 선택 모드로 들어가는 시간이 0.3초→**0.5초**로 늘었는데(의도된 완화), 테스트는 0.35초만 누르고 확인해 실패했다. 0.65초 누르도록 고쳤다.
- **설치 안내 자동 알림 7건** — 첫 방문에 설치 안내가 뜨던 것을 **일부러 없앴다**(검색으로 들어온 사람·검색엔진을 가리지 않도록, 다시 찾은 2번째 방문부터 안내). 테스트는 여전히 첫 방문 노출을 기대해 실패했다. 2번째 방문 상황으로 고치고, "첫 방문엔 안 뜬다"는 새 확인을 추가했다.

## 발견된 이슈

**앱 버그 0건.** 12건 모두 앱은 의도대로 동작했고 테스트만 낡아 있었다. 각 건은 실제 브라우저로 동작을 직접 확인한 뒤 수정했다.

근본 원인: e2e는 자동(CI)으로 안 돌리고 로컬에서만 돌려서, 코드가 바뀔 때 테스트가 조용히 뒤처졌다. 앞으로 기능을 바꾸면 관련 e2e도 함께 손봐야 한다.
