# 유닛 테스트 보고서: 검색 범위 지정(`in:책`) 해석 + 책 이름 찾기

**날짜:** 2026-06-22
**범위:** `js/app/search.js` — 검색창에서 입력한 `사랑 in:요한` 같은 "특정 책 안에서만" 조건을 떼어내는 `extractInScope`와 책 id를 한글 이름으로 바꾸는 `bookName`
**관련 PR:** (작성 중)

## 한 줄 요약

검색어에 `in:요한`처럼 책 범위를 적으면, 그 부분을 떼어내 "이 책들 안에서만 찾기"로 바꾸고 나머지 단어로 검색한다. 이 떼어내기 로직(검색 워커가 받기 전에 화면 쪽에서 먼저 처리하는 부분)과 책 이름 변환이 그동안 자동 검증 대상이 아니었는데, 이번에 점검하기 시작했다.

## 이 변경으로 무엇이 더 안전해졌는가

자동 검증 케이스 수가 **781건 → 792건**으로 늘었다 (+11, 회귀 0).

### `in:책` 범위 해석 (`extractInScope`)
- 범위 지정이 없으면 검색어 그대로, 범위 목록은 비움
- `사랑 in:요한` → "요한"을 책 범위로 떼어내고 검색어는 "사랑"만 남김
- `in:`이 검색어 앞에 와도 동작 (`in:gen 사랑`)
- `in:` 뒤 공백 허용 (`in: 요한`도 동일)
- 여러 `in:`은 누적 (여러 책 동시 범위)
- 책 약칭은 대소문자 무시 (`in:GEN` = `in:gen`)
- 해석되지 않는 책 이름은 검색어에 그대로 남겨 둠 (뒤이어 워커가 처리)
- 검색어 중복 공백은 하나로 정리

### 책 이름 찾기 (`bookName`)
- 등록된 id면 한글 이름으로 (예: `john` → "요한복음")
- 모르는 id는 id를 그대로 돌려줌 (안전 폴백)
- 이름표가 아직 안 준비됐어도(null) id 폴백 — 깨지지 않음

## 발견된 이슈

없음. 모든 경우가 의도대로 동작함을 확인. 추가로 실제 앱에서 `사랑 in:요한`을 검색해 범위가 정확히 요한복음으로 제한됨을 확인했다 (총 39건, 결과 전부 요한의 복음서, 오류 0건).
