# 유닛 테스트 보고서: 장 북마크 선택 삭제 picker

**날짜:** 2026-06-06
**범위:** `js/app/bookmark.js` — 읽기 화면 헤더에서 저장된 장의 북마크를 골라 지우는 새 선택 모달의 보조 계산
**관련 PR:** (작성 중)

## 한 줄 요약

이미 북마크된 장에서 헤더의 책갈피 아이콘을 누르면, 예전처럼 "이 장 북마크를 전부 지울까요?" 하고 묻는 대신 **이 장에 저장된 북마크들을 목록으로 보여주고 지울 것만 골라서** 지울 수 있게 바뀌었다. 그 화면의 "전체 선택" 체크 상태와 삭제 버튼 글자가 항상 맞게 표시되는지 자동으로 확인하기 시작했다.

## 무엇이 바뀌었나 (사용자 입장)

- 한 장에 북마크가 여러 개(예: 장 전체 + 1~3절 + 26~31절) 있을 때, 예전에는 책갈피를 끄면 **셋 다 한꺼번에** 사라졌다.
- 이제는 작은 목록 창이 떠서 **지울 항목만 체크**해 지운다. 고르지 않은 북마크는 그대로 남는다.
- 맨 위 **"전체 선택"** 한 번으로 모두 선택/해제할 수 있다.
- 아무것도 안 골랐을 때는 삭제 버튼이 회색(눌리지 않음)이고, 고르면 **"삭제 (2)"**처럼 고른 개수가 버튼에 적힌다 — 몇 개를 지우는지 누르기 전에 분명히 보인다.
- 데스크탑은 종전 그대로(헤더 책갈피 = 북마크 서랍 열기). 이 선택 창은 모바일 전용.

## 이 변경으로 무엇이 더 안전해졌는가

북마크 모듈 단독 자동 검증이 **133건** 통과(전체 회귀 **618건** 통과). 이번에 새로 자동 점검하는 동작:

### "전체 선택" 체크박스 상태 (`_selectAllState`)

목록에서 몇 개를 골랐느냐에 따라 맨 위 "전체 선택" 표시가 세 가지로 갈린다:

- 아무것도 안 골랐으면 → **빈 체크박스**
- 일부만 골랐으면 → **중간 상태(줄표)** — 전부도 아니고 아무것도 아닌 것도 아님을 한눈에
- 전부 골랐으면 → **꽉 찬 체크박스**
- 목록이 비어 있는(0개) 경우에도 오류 없이 "빈 체크박스"로 — 절대 중간 상태로 잘못 빠지지 않음

### 삭제 버튼 글자 (`_deleteBtnLabel`)

- 고른 게 없으면 → 그냥 **"삭제"** (이때 버튼은 비활성)
- 고른 게 있으면 → **"삭제 (1)" / "삭제 (3)"** 처럼 개수가 붙음

## 브라우저 동작 확인 (e2e)

자동 브라우저 시나리오 3종으로 실제 화면 흐름까지 확인:

- **전체 선택 후 삭제** — 책갈피 → 선택 창 등장(저장 창이 아님) → 처음엔 삭제 버튼 비활성 → "전체 선택" 체크 → 삭제 → 북마크 사라짐
- **취소하면 그대로** — 선택 창에서 취소하면 북마크가 남아 있음
- **고른 것만 삭제** — 한 장의 북마크 2개 중 하나만 체크해 삭제하면, 고른 것만 사라지고 **나머지는 유지**

화면도 라이트/다크 양쪽에서 눈으로 확인(목록·체크박스·삭제색·중간 상태 표시 정상).

## 발견된 이슈

없음. 위 모든 동작이 의도대로 동작함을 확인.

부수 정비: 모바일 헤더-토글 e2e가 iOS 환경에서 1.5초 뒤 자동으로 뜨는 "앱 설치 안내"에 가려 가끔 실패하던 문제(이번 변경 이전부터 있던 시점 의존성)를, 해당 테스트에서 설치 안내를 끄도록 고쳐 안정화했다.

## 다음 의제

선택 창의 **순수 계산 부분(전체 선택 상태·버튼 글자)은 자동 점검 대상**. 창 열고 닫기·포커스 트랩·체크박스 클릭 같은 DOM 동작은 e2e가 담당(ADR-013 dual-track 기조).
