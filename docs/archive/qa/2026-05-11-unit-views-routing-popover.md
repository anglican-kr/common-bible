# 유닛 테스트 보고서: 제목 옆 메뉴 + 스크롤 시 헤더 축소

**날짜:** 2026-05-11
**범위:** `js/app/views-routing.js` — 제목 클릭으로 펼치는 책 분류 / 장 선택 메뉴 + 스크롤에 따라 헤더가 작아지는 기능
**관련 PR:** (작성 중)

## 한 줄 요약

화면 위쪽 제목을 클릭하면 펼쳐지는 두 가지 메뉴(구약/외경/신약 분류 메뉴, 장 번호 격자) — 그리고 위로 스크롤할 때 자동으로 머리 부분이 작아져 본문 공간이 넓어지는 동작 — 이 의도대로 동작하는지 자동으로 확인하기 시작했다.

## 이 변경으로 무엇이 더 안전해졌는가

자동 검증 케이스 수가 **340건 → 366건**으로 늘었다 (+26).

이번에 자동으로 점검하게 된 동작:

### 책 분류 메뉴 (구약/외경/신약 고르기)

화면 위쪽 분류 제목을 누르면 아래로 펼쳐져 다른 분류로 바로 이동할 수 있는 메뉴.

- 메뉴는 처음에는 닫힌 상태로 시작
- 제목을 누르면 펼쳐지면서 첫 항목으로 자동 포커스 이동 (키보드 사용자도 바로 화살표·Tab으로 항목 이동 가능)
- 펼쳐진 상태에서 제목을 다시 누르면 닫힘
- 메뉴 바깥 아무 곳을 누르면 자동으로 닫힘
- 메뉴 안의 항목(분류 이름)을 누르면 닫히면서 해당 분류로 이동
- 현재 보고 있는 분류는 시각적으로 구별됨 (active 표시)
- 화면 낭독기 사용자에게 펼침/닫힘 상태가 정확히 안내됨 (`aria-expanded` true/false)

### 장 선택 메뉴 (예: 창세기 23장으로 점프)

책의 제목을 누르면 격자로 펼쳐지는 장 번호 메뉴.

- 책의 장 수만큼 정확한 개수의 장 번호가 격자에 표시 (예: 시편 → 150편, 창세기 → 50장)
- "머리말"이 있는 책(집회서)은 1장 앞에 머리말 링크가 추가로 표시
- 시편은 단위가 "편"으로, 그 외 책은 "장"으로 표시 (시편 23편 ≠ 시편 23장)
- 현재 보고 있는 장은 시각적으로 구별됨 (current 표시)
- 메뉴 옆에 "{책 이름} 목록으로" 뒤로가기 버튼과 북마크 버튼이 함께 배치됨
- 분류 메뉴와 동일한 펼침/닫힘 동작 (포커스 이동, 바깥 클릭으로 닫기, 항목 클릭으로 닫기)

### 스크롤에 따른 헤더 축소

본문을 읽기 위해 아래로 스크롤하면 머리 부분이 자동으로 작아져 본문이 더 잘 보임. 다시 위로 올라가면 원래대로 돌아옴.

- 60픽셀 이상 내려가면 머리 부분이 작아짐
- 10픽셀 이내로 다시 올라오면 원래 크기로 복귀
- 그 사이 구간(10~60픽셀)에서는 깜빡거리지 않고 현재 상태 유지 — 즉, 살짝 위아래로 흔들어도 머리 부분이 떨었다 줄었다 반복하지 않음
- 정확히 60픽셀이거나 정확히 10픽셀일 때는 안전하게 현재 상태 유지 (경계값에서 흔들림 방지)
- 스크롤 자체를 막지 않는 가벼운 감지 (passive 리스너 — 모바일 터치 스크롤이 매끄럽게 유지됨)

## 다음 의제

이번 PR로 **`views-routing.js`의 검증 가능한 영역은 거의 모두 자동 점검 대상**. 남은 미커버:
- **본문 위로 당겨 새로 고침** (Pull-to-refresh) — 손가락 터치 동작 + 트랜지션 애니메이션
- **읽은 위치 자동 추적** (스크롤하면서 보이는 절을 기억) — 정확한 위치 계산 필요
- **오디오 재생기** — 미디어 이벤트·진행 표시줄 등

위 세 영역은 jsdom 도입(별도 PR)에서 다룰 의제.

다른 모듈 우선:
- **외관 설정 화면** (`settings-ui.js`)
- **설치 안내 모달** (`install.js`)
- **북마크·검색 화면**의 UI 영역
