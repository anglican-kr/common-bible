# ADR-027: 디자인 시스템 토큰

- 일시: 2026-06-01
- 상태: 승인됨 — 구현 중 (토큰 정의 + 노트/캘린더 적용, release/1.6.0)
- 관련 ADR: ADR-024(목록 탭·헤더), ADR-025(헤더 elevation), ADR-026(노트·하단 내비)

## 맥락

색 토큰(`--bg`/`--bg-card`/`--text`/`--accent`/`--border`/`--radius` + 색 스킴)은
있었지만 **스페이싱·타이포·반경·그림자에 스케일이 없어** 화면마다 즉흥적인 값이
쌓였다. `css/style.css` 실측 분포:

- 스페이싱(padding/margin/gap): `0.5/1/0.4/0.75/0.6/0.3/1.25/0.25/0.8/0.45/0.35/0.2…rem`
  등 20종 이상 난립 — 4px 그리드가 없어 리듬이 깨졌다.
- 폰트 크기: `0.88/0.9/0.85/1/0.95/0.75/1.1/0.8/0.82/0.78…rem` 등 — 위계 흐릿.
- 반경: `var(--radius)`(8px) 지배적이나 `4px/16px/6px/999px/2px/14px/5px` 하드코딩 혼재.
- 그림자: 매번 다른 `rgba` 조합(오버레이·elevation·focus ring 구분 없음).
- surface 계층 부족: `--bg`/`--bg-card`/`--accent`만 있어 섹션·hover·muted 표현이 빈약.

결과적으로 "디자인 감각"보다 **일관성**의 문제였다.

## 결정

기존 값을 **역설계**해 최빈값을 흡수하는 스케일을 `css/style.css :root`에 토큰으로
정의한다(추측이 아니라 실측 분포 기반 → 기존 규칙이 최소 변경으로 흡수). 기존 토큰
(`--bg`, `--radius`, `--shadow` 등)은 호환을 위해 **유지**하고 그 위에 얹는다.

- **스페이싱** `--sp-0…7` — 4px(0.25rem) 그리드(2/4/8/12/16/24/32/48px).
- **타이포** `--fs-xs/sm/base/md/lg/xl` + `--lh-tight/normal/loose` — base 1rem,
  `--fs-sm`(0.875rem)이 본문 보조 최빈값을 대표.
- **반경** `--r-sm`(4)/`--r-md`(=`--radius` 8)/`--r-lg`(16)/`--r-pill`.
- **elevation** `--shadow-1/2/3` + `--focus-ring`. 다크 테마는 알파를 키운 오버라이드.
- **surface/상태** `--bg-subtle`(섹션·hover 채움), `--text-muted`(3차 텍스트),
  `--hover-overlay`/`--press-overlay`(중립 hover/press 워시). 다크 변형 포함.

스케일·타이포·반경 토큰은 테마 독립(`:root`만), surface·그림자·오버레이는
`[data-theme="dark"]` 에 오버라이드한다. `index.html` 의 critical CSS `:root` 는
프리페인트용 최소 토큰만 두고 건드리지 않는다.

## 적용 전략 — 점진

토큰만 먼저 깔고 화면별로 옮긴다. 신규/즉흥 비중이 큰 **노트·캘린더(ADR-026)부터
토큰으로 리핏**해 효과를 즉시 확인한다. 기존 화면(읽기·검색·북마크·설정)은
후속에서 같은 토큰을 참조하도록 이관한다. 버튼/칩/시트 등 컴포넌트 프리미티브
표준화도 후속(이번 범위는 토큰 + 노트/캘린더 적용까지).

## 대안

- **외부 CSS 프레임워크/디자인 시스템 도입**: 무프레임워크 원칙(ADR-001)·번들·CSP에
  반해 기각. 자체 토큰으로 충분.
- **임의의 이상적 스케일을 새로 정의**: 기존 수백 곳을 전부 수정해야 해 회귀 위험 큼.
  실측 역설계로 흡수 비용 최소화.
- **한 번에 전 화면 리팩터**: 검증 면적이 너무 넓음. 토큰 우선 + 점진 적용.

## 영향

- `css/style.css`: `:root`/`[data-theme="dark"]` 토큰 추가, 노트·캘린더·피커 규칙을
  토큰 기반으로 리핏(값 동등 또는 한 스텝 내 흡수).
- 후속: 읽기·검색·북마크·설정 화면 토큰 이관, 컴포넌트 프리미티브(버튼/칩/시트) 표준화.
- 데이터·JS 동작·라우트 불변. 시각적 미세 변화만(스페이싱·폰트 정렬).
