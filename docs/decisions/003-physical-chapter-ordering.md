## 결정 요약

원본 텍스트에 등장하는 절 표기 패턴을 아래와 같이 처리한다.

| 패턴 | 예시 | 처리 방식 |
|------|------|-----------|
| 물리적 장 재배치 | `아모 6:9` (6장 내 순서 변경) | 원문 물리적 순서 보존 |
| 다른 장 절 삽입 | `이사 41:6` (40장 중간 삽입) | 삽입된 장에 포함, `chapter_ref` 표기 |
| 단락 연속 (¶ 또는 빈 줄) | `¶ 시드키야가...` | 앞 절 text에 이어붙임, `has_paragraph=true` |
| 절 범위 | `17-18 거기에서...` | `range_end` 필드로 끝 절 번호 저장 |
| 부분절 | `2a 그가...` | `part` 필드로 분할 식별자 저장 |
| 사본 이중 번호 | `1_2 모르드개는...` | 앞 번호를 기준 절 번호로, 뒷 번호를 `alt_ref`로 저장 |

## v1 — 물리적 장 처리 (2026-03-26)

### 맥락

공동번역성서는 성서학자들이 히브리어·그리스어 사본을 검토하여 일부 절의 위치를 재배치한 번역이다.
원본 텍스트의 형식은 두 가지 패턴으로 나타난다.

**같은 장 내 절 재배치** (48개 장에서 발생):
```
아모 6:1  ...
아모 6:11 ...  ← 11절이 1절 직후에 등장
아모 6:9  ...  ← 9절이 그 뒤에 등장
10 ...
```

**다른 장의 절 삽입** (6곳):
```
이사 40:19 ...
이사 41:6  ...  ← 41장 6절이 40장 본문 중간에 삽입
7 ...
이사 40:20 ...  ← 다시 40장으로 복귀
```

발생 위치:
- 이사야 40장 ← 41:6절
- 잠언 5장 ← 6:22절
- 호세아 14장 ← 13:14절
- 역대기상 6장 ← 9:33절
- 욥기 25장 ← 26:5절
- 욥기 27장 ← 176:18절 (원문 표기 오류로 추정)

### 검토한 대안

- **A. 물리적 장 (채택)**: 원문 순서 그대로 저장. 다른 장의 절은 `chapter_ref`로 원래 장 번호 표기.
- **B. 정경적 장**: 표기된 장 번호에 귀속. 학자들의 읽기 순서 훼손.
- **C. 양쪽 장 중복 저장**: 데이터 중복, 정본 불명확.

### 채택 이유

공동번역성서의 절 재배치는 **성서학자들의 의도적인 학문적 판단**이다.
원문의 읽기 순서를 그대로 보존하는 것이 번역본을 충실히 전달하는 취지에 부합한다.

### 새 장 시작 조건 (parser.py)

```
is_new_chapter = (
    현재 장이 없음
    OR 책이 바뀜
    OR (해당 책+장 조합이 아직 열리지 않음 AND 절 번호 == 1)
)
```

- `아모 6:9` → 아모스 6장이 이미 열려 있으므로 새 장 아님
- `이사 41:6` → 41장 미개방이지만 절 번호가 1이 아니므로 새 장 아님, 40장에 삽입
- `이사 41:1` → 41장 미개방 + 절 번호 1 → 새 장 시작

## v2 — 절 번호 패턴 4종 추가 (2026-03-27)

### 맥락

원본 텍스트를 전수 조사한 결과, 파서가 처리하지 못하고 조용히 누락시키는 절 표기 패턴 4종이 추가로 발견됐다.

### 패턴 1: 단락 연속 (¶ 또는 빈 줄)

같은 절 안에서 단락이 나뉘는 경우. 두 가지 형태로 나타난다.

**¶ 접두 연속** (59곳):
```
20 예루살렘과 유다는 야훼의 진노를 사 마침내 그 앞에서 쫓겨나고 말았다.
¶ 시드키야가 바빌론 왕에게 반기를 들었다.
```

**빈 줄 연속** (8곳 — 출애굽기 2곳, 이사야 4곳, 오바댜 1곳, 요한복음 1곳):
```
13 너희는 내가 너희에게 말한 모든 것을 명심하여라.

다른 신들의 이름을 찬양하지 마라. 그런 소리는 입 밖에도 내지 마라.
14 일 년에 세 차례 내 앞에서 축제를 올려라.
```

**처리**: 앞 절 `text`에 `\n` + 연속 줄 이어붙임, `has_paragraph = true`.
프론트엔드는 `\n`을 기준으로 같은 단락(`<p>`) 안에서 줄 바꿈 처리.

### 패턴 2: 절 범위 (16곳)

두 개 이상의 절이 하나로 합쳐진 경우.

```
17-18 거기에서 북쪽으로 방향을 바꾸어 엔세메스로 빠진다...
```

**처리**: 첫 번째 절 번호를 `number`로, 마지막 절 번호를 `range_end`로 저장.
검색: 범위 내 모든 절 번호(17, 18)에 인덱싱.

### 패턴 3: 부분절 (24곳)

성서학자들이 절을 분할하여 각기 다른 위치로 이동시킨 경우. 판관기·사무엘기·욥기·시편·이사야서 등에서 발생.

```
2a 그가 자기 어머니에게 말하였다...
3b 여기 있으니 받아주십시오.
3a 아들은 돈 천백 냥을 어머니에게 돌려드렸다...
```

**처리**: 절 번호를 `number`로, 분할 식별자(`"a"`, `"b"`)를 `part`로 저장.
검색: `"2a"`, `"3b"` 형태로 인덱싱.

### 패턴 4: 사본 이중 번호 (116곳)

에스더서와 다니엘서의 그리스어 부가문. 칠십인역(LXX)과 히브리어 사본의 절 번호가 달라 두 번호를 함께 표기.

```
에스 1:1_1 아하스에로스 대왕 제이년...
1_2 모르드개는 수사에 사는...
96_29 이제 나는 영을 내린다...
```

**처리**: 밑줄 앞 번호(LXX)를 `number`로, 밑줄 뒤 번호(히브리어 사본)를 `alt_ref`로 저장.
렌더링: 뒷 번호를 절 번호 옆에 위첨자로 표시 (예: 96²⁹).
해당 장 JSON에 `has_dual_numbering: true` 플래그 추가 → 푸터에 번호 체계 설명 표시.
검색: `number`(앞 번호) 기준.

## 최종 데이터 모델

### Verse 데이터클래스

```python
@dataclass
class Verse:
    number: int
    text: str
    has_paragraph: bool = False
    chapter_ref: Optional[int] = None  # 다른 장의 절 삽입 시 원래 장 번호
    range_end: Optional[int] = None    # 절 범위 (예: 17-18 → range_end=18)
    part: Optional[str] = None         # 부분절 식별자 (예: "a", "b")
    alt_ref: Optional[int] = None      # 사본 이중 번호의 히브리어 사본 절 번호
```

모든 Optional 필드는 값이 있을 때만 JSON에 포함 (null 생략).

### 장별 JSON 예시

```json
{ "number": 17, "range_end": 18, "text": "...", "has_paragraph": false }
{ "number": 2, "part": "a", "text": "...", "has_paragraph": false }
{ "number": 96, "alt_ref": 29, "text": "...", "has_paragraph": false }
{ "number": 6, "chapter_ref": 41, "text": "...", "has_paragraph": false }
```

### 사본 이중 번호 장 JSON 플래그

```json
{
  "book_id": "esth",
  "chapter": 1,
  "has_dual_numbering": true,
  "verses": [...]
}
```
