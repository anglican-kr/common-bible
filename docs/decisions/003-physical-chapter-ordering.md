# ADR-003: 물리적 장(Physical Chapter) 처리 방식

- 일시: 2026-03-26
- 상태: 승인됨

## 결정

성경 데이터를 장별 JSON으로 분리할 때, 절의 위치를 원문 텍스트의 물리적 순서 기준으로 결정한다.
다른 장의 절이 삽입된 경우 해당 절이 실제로 등장하는 장 파일에 포함하며, `chapter_ref` 필드로 원래 장 번호를 표기한다.

## 맥락

공동번역성서는 성서학자들이 히브리어·그리스어 사본을 검토하여 일부 절의 위치를 재배치한 번역이다.
원본 텍스트의 형식은 두 가지 패턴으로 나타난다.

**같은 장 내 절 재배치** (48개 장에서 발생):
```
아모 6:1  ...
아모 6:11 ...  ← 11절이 1절 직후에 등장
아모 6:9  ...  ← 9절이 그 뒤에 등장
10 ...
```

**다른 장의 절 삽입** (6곳):
```
이사 40:19 ...
이사 41:6  ...  ← 41장 6절이 40장 본문 중간에 삽입
7 ...
이사 40:20 ...  ← 다시 40장으로 복귀
```

발생 위치:
- 이사야 40장 ← 41:6절
- 잠언 5장 ← 6:22절
- 호세아 14장 ← 13:14절
- 역대기상 6장 ← 9:33절
- 욥기 25장 ← 26:5절
- 욥기 27장 ← 176:18절 (원문 표기 오류로 추정)

## 검토한 대안

### A. 물리적 장 (채택)
- 원문에서 해당 구간에 나타나는 순서대로 저장
- 다른 장의 절은 `chapter_ref` 필드로 원래 장 번호 표기
- 장점: 학자들이 의도한 읽기 순서 완전 보존
- 단점: 일부 장에서 특정 절 번호가 빠지는 것처럼 보일 수 있음
  (예: 이사야 41장에서 6~7절이 없는 것처럼 보임 — 실제로는 40장에 있음)

### B. 정경적 장
- 각 절을 표기된 장 번호에 귀속하여 저장
- 장점: 장별 내비게이션이 표준 성경과 일치
- 단점: 학자들이 의도한 읽기 순서가 훼손됨

### C. 물리적 장 + 양쪽 장에 중복 저장
- 장점: 어느 장으로 접근해도 해당 절을 볼 수 있음
- 단점: 데이터 중복, 어느 것이 정본인지 불명확

## 채택 이유

공동번역성서의 절 재배치는 번역상의 실수가 아니라 **성서학자들의 의도적인 학문적 판단**이다.
성경 읽기 앱의 목적은 이 번역본을 충실히 전달하는 것이므로, 원문의 읽기 순서를 그대로 보존하는 것이 취지에 부합한다.
정경적 장 번호 체계에 맞추면 학자들이 복원한 본문의 흐름이 사라진다.

## 결과

### 새 장 시작 조건 (parser.py)

```
is_new_chapter = (
    현재 장이 없음
    OR 책이 바뀜
    OR (해당 책+장 조합이 아직 열리지 않음 AND 절 번호 == 1)
)
```

이 조건으로:
- `아모 6:9` → 아모스 6장이 이미 열려 있으므로 새 장 아님, 동일 장의 절로 처리
- `이사 41:6` → 이사야 41장 미개방이지만 절 번호가 1이 아니므로 새 장 아님, 40장에 삽입
- `이사 41:1` → 이사야 41장 미개방 + 절 번호 1 → 새 장 시작

### 데이터 모델

`Verse` 데이터클래스에 `chapter_ref: Optional[int]` 필드 추가:

```python
@dataclass
class Verse:
    number: int
    text: str
    has_paragraph: bool = False
    chapter_ref: Optional[int] = None  # 다른 장의 절일 때만 설정
```

장별 JSON에서 `chapter_ref`는 값이 있을 때만 포함 (null 생략):

```json
{ "number": 6, "chapter_ref": 41, "text": "...", "has_paragraph": false }
```

### 프론트엔드 동작 지침

- `chapter_ref`가 없는 절: 현재 장의 절로 표시
- `chapter_ref`가 있는 절: 해당 절 번호 옆에 원래 장 번호를 작게 표기
  (예: 이사야 40장 읽기 중 `⁴¹⁶ 끼리끼리 손발이 맞아...`)
- 검색 기능에서 해당 절은 실제 파일이 위치한 장 기준으로 인덱싱
