# ADR-006: 운문 원본 텍스트 포맷 규칙

## 개정 이력

| 버전 | 일시       | 내용                                                           |

## 결정 요약

운문(시) 본문의 반행(hemistich) 줄바꿈과 스탠자(연) 구분을 원본 텍스트에서 직접 표현하고, 파서가 이를 인식하여 구조화된 데이터로 변환한다.

| 요소 | 형식 | 예시 |
|------|------|------|
| 반행 연속 | 2칸 들여쓰기 (`  `) | `  악을 꾸미는 자리에...` |
| 단일 반행(monocolon) | 반행 없이 절 번호 행만 | (렌더러가 문맥으로 자동 판별) |
| 스탠자 구분 | 빈 줄 1개 | (절 그룹 사이 빈 줄) |
| 장 구분 | 빈 줄 2개 | (기존 규칙, 변경 없음) |
| 산문 단락 | `¶` | (기존 규칙, 변경 없음) |
| 절 내 산문→운문 전환 | 산문 행 + 빈 줄 + 2칸 들여쓰기 | (창세 4:23 참고) |
| 절 내 운문→산문 전환 | 2칸 들여쓰기 + 빈 줄 + 산문 행 | |

---

## 맥락

공동번역성서의 시편, 잠언, 전도서, 아가, 애가 등 운문 본문은 현재 산문과 동일한 형식으로 저장되어 있다. 각 절이 한 줄에 평문으로 기록되어 있어, 히브리 시의 핵심 구조인 대구(parallelism)와 연(stanza) 구분이 소실된 상태다.

```
시편 1:1 복되어라. 악을 꾸미는 자리에 가지 아니하고 죄인들의 길을 거닐지 아니하며 조소하는 자들과 어울리지 아니하고,
2 야훼께서 주신 법을 낙으로 삼아 밤낮으로 그 법을 되새기는 사람.
```

인쇄본 성경에서는 운문을 행 단위로 배치하여 대구 구조를 시각적으로 표현한다. 이 구조를 원본 텍스트에 반영하면, 데이터 파이프라인 전체(파서 → JSON → 렌더러)가 운문을 산문과 구별하여 처리할 수 있다.

### 히브리 시의 기본 구조

히브리 운문의 기본 단위는 **반행(半行, hemistich)**이다. 한 절이 보통 2~3개의 반행으로 이루어지며, 각 반행은 의미적 대구를 이룬다.

```
야훼여! 나를 괴롭히는 자 왜 이리 많사옵니까?    ← 전반행
나를 넘어뜨리려는 자 왜 이리 많사옵니까?         ← 후반행 (동의적 대구)
```

여러 절이 모여 하나의 **스탠자(연)**를 구성하며, 스탠자 사이에는 주제나 화자의 전환이 일어난다.

## 검토한 대안

### A. 원본 텍스트에 포맷 추가 (채택)

원본 텍스트에 들여쓰기와 빈 줄로 운문 구조를 직접 표현한다.

- 장점: 원본이 single source of truth. 인쇄본과 대조하며 편집 가능. 파서가 구조를 자연스럽게 추출.
- 단점: 수백 장의 수작업 편집 필요.

### B. 책 단위 플래그 + 렌더러 처리

`books.json`에 `"kind": "poetry"` 플래그를 추가하고, 렌더러에서 절을 block 요소로 표시한다.

- 장점: 데이터 변경 최소. 즉시 구현 가능.
- 단점: 반행 구분 불가. 스탠자 구분 불가. 혼합 책(욥기 등) 처리 불가.

### C. 별도 포맷 파일

운문 구조를 담은 보조 파일(예: `poetry-format.json`)을 만들어 책/장/절별 반행 위치를 기록한다.

- 장점: 원본 텍스트 수정 없음.
- 단점: 두 개의 source of truth 관리 필요. 원본과 보조 파일 간 동기화 문제.

### 채택 이유

원본 텍스트가 데이터 파이프라인의 시작점이므로, 운문 구조를 원본에 직접 포함하는 것이 가장 일관된 접근이다. 인쇄본을 참고하여 반행 단위로 편집할 수 있어 정확도를 검증하기 쉽다.

## 원본 텍스트 포맷 규칙

### 규칙 1: 반행 연속 — 2칸 들여쓰기

절의 첫 반행은 기존처럼 절 번호로 시작한다. 이후 반행은 **2칸 스페이스**로 들여쓰기한다.

```
시편 24:1 이 세상과, 그 안에 가득한 것이 모두 야훼의 것,
  이 땅과 그 위에 사는 것이 모두 야훼의 것,
2 주께서 바다 밑에 기둥을 박으시고
  이 땅을 그 물 위에 든든히 세우셨다.
```

- 2칸 들여쓰기로 시작하고 절 번호가 없는 줄 = 이전 절의 다음 반행
- 한 절에 반행이 3개 이상일 수 있다 (3행 대구)

### 규칙 2: 스탠자(연) 구분 — 빈 줄 1개

스탠자 사이에 빈 줄 1개를 삽입한다. 기존 장 구분(빈 줄 2개)과 구별된다.

**절 사이 스탠자 구분** — 빈 줄 뒤에 절 번호가 오는 경우:

```
6 이런 사람이 하느님을 찾는 사람이며
  야곱의 하느님 앞에 나아갈 사람이다. (셀라)

7 문들아, 머리를 들어라.
  오래된 문들아, 일어서라.
  영광의 왕께서 드신다.
```

**절 내부 스탠자 구분** — 빈 줄 뒤에 들여쓰기가 오는 경우:

```
5 야훼께서 심판하실 때에
  머리조차 들지 못하고,

  죄인이라 의인들 모임에
  끼지도 못하리라.
```

파서는 빈 줄 다음에 오는 줄의 형태로 구분한다:

| 빈 줄 뒤                   | 해석                | 데이터 표현                    |
| -------------------------- | ------------------- | ------------------------------ |
| 절 번호 (`7 문들아...`)    | 절 사이 스탠자 구분 | 다음 절에 `stanza_break: true` |
| 들여쓰기 (`  죄인이라...`) | 절 내부 스탠자 구분 | 같은 절 텍스트에 `\n\n` 삽입   |

### 규칙 3: 기존 규칙과의 공존

| 구분        | 빈 줄 수 | 용도                |
| ----------- | -------- | ------------------- |
| 스탠자 구분 | 1개      | 운문 연 사이        |
| 장 구분     | 2개      | 장과 장 사이 (기존) |

산문의 `¶` 표기와 운문의 들여쓰기는 상호 배타적이다. 한 책 안에서 산문 구간은 `¶`를 사용하고, 운문 구간은 들여쓰기를 사용한다.

### 규칙 4: 산문·운문 혼합(절 단위) — 들여쓰기 유무로 자동 구분

욥기처럼 산문과 운문이 한 장 안에서 전환되는 경우, 절 단위로 들여쓰기 유무가 곧 구분자가 된다. 별도의 전환 마커는 필요 없다.

```
욥기 3:1 ¶ 마침내 욥이 먼저 입을 열어 자기의 생일을 저주하며    ← 산문 (들여쓰기 없음)
2 부르짖었다.                                                    ← 산문
3 내가 태어난 날이여,                                            ← 운문 시작 (들여쓰기 있음)
  차라리 사라져버려라.
  사내아이를 배었다고 하던 그 밤도
  사라져버려라.
```

```
욥기 42:5 당신께서 어떤 분이시라는 것을                          ← 운문
  소문으로 겨우 들었었는데,
  이제 저는 이 눈으로 당신을 뵈었습니다.
6 그리하여 제 말이 잘못되었음을 깨닫고                           ← 운문
  티끌과 잿더미에 앉아 뉘우칩니다.
7 ¶ 야훼께서 욥과 말씀을 마치신 다음에 데만 사람 엘리바즈에게... ← 산문 복귀
```

산문 복귀의 신호: `¶` 표기. 렌더러는 `has_paragraph=true` 절을 만나면 운문 문맥을 종료한다.

### 규칙 5: 단일 반행(monocolon) — 렌더러 문맥 자동 판별

히브리어 원문에는 짝이 없는 단독 행(monocolon)이 존재한다. 이 절은 들여쓰기 연속행이 없어 파서는 `text`에 `\n`을 생성하지 않는다. 원본 텍스트에 별도 마커를 추가하지 않는다.

렌더러가 스탠자 문맥(`inPoetryStanza`)으로 자동 처리한다:

- 운문 절(`\n` 포함)을 만나면 `inPoetryStanza = true` 진입
- `has_paragraph=true` 절을 만나면 `inPoetryStanza = false` 종료
- `inPoetryStanza=true`인 절은 단일행이라도 운문으로 렌더링

```
4 나 야훼께 부르짖으면
  당신의 거룩한 산에서 들어주십니다. (셀라)   ← 운문 → inPoetryStanza=true
5 자리에 들면 자나깨나...                      ← monocolon, inPoetryStanza=true → 운문
6 적들이 밀려와 에워쌀지라도...               ← monocolon, inPoetryStanza=true → 운문
7 야훼여, 일어나소서.
  나의 하느님, 구하여 주소서.                  ← 운문
```

### 규칙 6: 절 내 산문+운문 혼합

한 절 안에서 산문과 운문이 공존하는 경우(창세 4:23 등). 순서에 따라 두 가지 패턴으로 표기한다.

**패턴 A — 산문 → 운문**: 절 번호 행에 산문, 빈 줄 1개, 이후 운문을 2칸 들여쓰기

```
창세 4:23 라멕이 아내들에게 말하였다.

  아다야, 실라야, 내 말을 들어라.
  라멕의 아내들아, 내 말에 귀를 기울여라.

  나를 해치지 마라. 죽여버리리라.
  젊었다고 하여 손찌검하지 마라. 죽여버리리라.
```

파서: 빈 줄 + 2칸 들여쓰기 = 절 내부 스탠자 구분(`\n\n`). 결과 텍스트에 `\n` 포함 → 운문 판별.

**패턴 B — 운문 → 산문**: 운문을 2칸 들여쓰기, 빈 줄 1개, 산문은 들여쓰기 없음

```
  운문 행

산문 행
```

파서: 2칸 들여쓰기 → `\n` 추가, 빈 줄 + 산문 행 → 단락 연속(`\n`, `has_paragraph=true`).
렌더러: `\n` 포함이므로 `isPoetry=true`. 산문 행도 블록으로 표시된다.

## 전체 예시: 시편 1편

### 변경 전

```
시편 1:1 복되어라. 악을 꾸미는 자리에 가지 아니하고 죄인들의 길을 거닐지 아니하며 조소하는 자들과 어울리지 아니하고,
2 야훼께서 주신 법을 낙으로 삼아 밤낮으로 그 법을 되새기는 사람.
3 그에게 안 될 일이 무엇이랴! 냇가에 심어진 나무 같아서 그 잎사귀가 시들지 아니하고 제 철 따라 열매 맺으리.
4 사악한 자는 그렇지 아니하니 바람에 까불리는 겨와도 같아.
5 야훼께서 심판하실 때에 머리조차 들지 못하고, 죄인이라 의인들 모임에 끼지도 못하리라.
6 악한 자의 길은 멸망에 이르나, 의인의 길은 야훼께서 보살피신다.
```

## 파서 변경 사항 (완료)

1. **2칸 들여쓰기 줄 인식**: 이전 절의 `text`에 `\n` + 텍스트 이어붙이기 (`has_paragraph` 설정 안 함)
2. **장 내 빈 줄 1개 인식**: 빈 줄 다음 줄의 형태에 따라 분기
   - 다음 줄이 절 번호 → 해당 절에 `stanza_break: true`
   - 다음 줄이 들여쓰기 → 현재 절 `text`에 `\n\n` + 텍스트 이어붙이기 (절 내부 스탠자)
   - 다음 줄이 일반 텍스트 → ADR-003 단락 연속 (`has_paragraph=true`)
3. **`Verse` 데이터클래스에 `stanza_break: bool = False` 필드 추가**
4. **단일 반행(monocolon)**: 파서 변경 없음. 렌더러의 `inPoetryStanza` 문맥으로 처리

### 렌더러의 운문 판별 규칙

| 조건 | 판별 | 렌더링 |
|------|------|--------|
| `text`에 `\n` 포함 | 운문 절 → `inPoetryStanza=true` 진입 | block, 반행 단위 표시 |
| `text`에 `\n\n` 포함 | 운문 절 + 절 내부 스탠자 | block + 스탠자 간격 |
| `text`에 `\n` 없음 + `inPoetryStanza=true` | monocolon | block (운문 문맥 유지) |
| `text`에 `\n` 없음 + `inPoetryStanza=false` | 산문 절 | inline 표시 |
| `has_paragraph=true` | 산문 → `inPoetryStanza=false` 종료 | 단락 구분 후 inline |
