# ADR-037: 감사성찬례 전례독서 데이터 모델 · 교회력 계산 엔진

- 일시: 2026-07-04
- 상태: **승인됨 — 구현 중** (D1 전례시편 · D2 독서 데이터 · D3 연중 구간표 완료 2026-07-13 · **A1 엔진 남음**)
- 관련: ADR-036(교회력 데이터 모델·본기도 — 좌표 체계·computus 기준의 권위 출처), ADR-035(절 부분집합·연속 병합 렌더 — 독서 본문 렌더 기반), ADR-010(북마크 verseSpec 문법), ADR-021(콘텐츠 해시 매니페스트·SW 캐시), ADR-013(유닛 테스트 하네스), ADR-038(캘린더·독서 뷰·검색 UI), ADR-039(전례시편 DSL·파서·`lps` 스키마 — 데이터 저장소, D1 권위 출처)

## 배경 — 이 ADR이 다루는 범위

주일/주간·축일·재일 **감사성찬례 독서**(제1독서·시편·제2독서·복음)와 본기도를 날짜별로 조회·봉독하는 기능의 **데이터 모델과 계산 엔진**을 정한다. UI(캘린더·독서 뷰·검색)는 ADR-038.

원료는 서울교구 홈페이지 스크랩(`common-bible-data/liturgical/` — 838 기사, `articles.json`의 `readings[]` + html 캐시의 독서 전문). ADR-036이 본기도 데이터 모델을 정했고, 그 §2에서 "독서(lectionary) 축은 별도 `eucharist-readings.json` + 교회력 엔진 소관"으로 미룬 부분을 본 ADR이 채운다.

렌더 기반은 이미 있다 — ADR-035의 `appendVerses`(절 부분집합 렌더)와 인접 연속 병합은 로드맵 메모에 "교회력 계산 결과를 입력으로 바꾸면 같은 렌더 경로를 재사용"이라고 명시해 두었다. 본 ADR은 그 입력(데이터·엔진)을 정의한다.

## 결정

### 1. `eucharist-readings.json` — 좌표 스키마 평면 배열 (데이터 저장소 `lectionary/`)

**한 레코드 = (교회력일 × 독서 트랙 × 대안 세트).** ADR-036 §1과 같은 평면 배열·자기 좌표 원칙. 본기도와 달리 **요일·주기 축 병합을 하지 않는다** — 독서는 해마다·요일마다 실제로 다르다.

```json
{
  "_meta": { "source": "seoul.anglican.kr 감사성찬례-전례독서 태그", "articles": 838 },
  "_vocab": { "…": "ADR-036 §4와 동일 + slot" },
  "entries": [
    {
      "id": "ordinary-10-sun-A-t1-s1",
      "kind": "temporal", "season": "ordinary", "week": 10,
      "type": "sunday", "weekday": null, "year": "A",
      "date": null, "lunar": null, "name": null, "aliases": null,
      "title": "연중10주",
      "reading_track": 1,
      "set_no": 1, "set_total": 1, "set_note": null,
      "readings": [
        { "slot": "first",  "bookId": "gen",  "refs": [{ "chapter": 12, "verseSpec": "1-9" }], "label": "창세 12:1-9" },
        { "slot": "psalm",  "bookId": "lps",  "refs": [{ "chapter": 33, "verseSpec": "1-12" }], "label": "시편 33:1-12" },
        { "slot": "second", "bookId": "rom",  "refs": [{ "chapter": 4, "verseSpec": "13-25" }], "label": "로마 4:13-25" },
        { "slot": "gospel", "bookId": "matt", "refs": [{ "chapter": 9, "verseSpec": "9-13,18-26" }], "label": "마태 9:9-13, 18-26" }
      ],
      "_sources": [{ "id": "26662" }]
    }
  ]
}
```

- **`id` = 좌표 기반 안정 슬러그**(재생성해도 동일). ADR-036 §10의 "잠정 순번 id를 키로 쓰지 말 것"의 해법 — 검색 결과·URL(`/lectionary/d/<slug>`)이 이 id에 착지한다.
- **`refs` 배열이 장 경계를 표현한다.** `1고린 3:18-4:5` → `[{chapter:3, verseSpec:"18-23"}, {chapter:4, verseSpec:"1-5"}]`. 열린 끝은 빌드 시 실제 장 JSON의 최대 절로 닫는다(검증 겸용). 장 경계 병합 렌더는 ADR-035 그대로.
- **`verseSpec` 문법은 북마크(ADR-010)와 완전 동일** — `"all" | "3" | "1-17" | "1-5,10-15" | "3a"`. 반절 표기 `상→a`, `하→b`. 범위 안 반절은 분해: `"46하-55"` → `"46b,47-55"`, 범위 끝이 `"2:4상"`이면 그 장의 verseSpec은 `"1-3,4a"`로 분해된다.
- **대안 세트 explode**: `(또는 …)`·`또는` 인라인 대안·괄호 선택 확장(`9:35-10:8(9-23)`)이 있으면 **완전한 세트를 통째로 복제**(원본 세트 + 대안 적용 세트, `set_no`/`set_total`). 조건부 대안(`(주의 변모 주일로 지키는 경우, …)`)은 별도 세트 + `set_note`에 조건 원문. 뷰는 항상 "완전한 세트 하나"를 그리고 세트 전환은 UI 몫(ADR-038).
- **시편 슬롯**: 기본 `bookId:"lps"`(전례시편 책, §3). 송가 대체는 `canticle` id(`magnificat` 등) + lps 부록 장 참조 + `scripture_ref`(출처 성구) 병기 — 표기 시 시편 장/절 옆에 송가 명칭을 함께 쓴다(ADR-038). 시편 슬롯의 비시편 본문(다니엘·요나·토비트)은 `slot:"psalm"` 유지 + 일반 성서 bookId.
- **본기도 연결은 필드가 아니라 좌표 조인.** 같은 좌표로 `eucharist-collects` 를 ADR-036 §2·§4 폴백 규칙(weekday/year `null`=공통, 배열=포함 매치)으로 조회. 송영은 ADR-036 §3 개정(2026-07-04)대로 분리 저장·앱 상수 3종(시편 끝의 영광송과 별개 용어).
- **파일 배치: 단일 파일** `lectionary/eucharist-readings.json` (~300–400KB 예상 — `search-dc.json`보다 작고, 캘린더·독서 뷰가 전체를 쓰므로 분할 이득 없음).

> **개정 (2026-07-13, D2 구현 — data#12):** 844기사 → **906 엔트리**, 미해석 문자열 0건. 구현하며 확정·수정한 것들.
>
> **① id 슬러그 규칙** (ADR 예시 `ordinary-10-sun-A-t1-s1` 에서 역산 + 미명세 구간 확정) — 좌표만의 순수 함수다.
>
> | 좌표 | 슬러그 |
> | --- | --- |
> | 절기 격자 | `{season}-{week}-{sun\|요일코드\|feast\|fast}[-{고유명}]-{year}-t{track}-s{set}` |
> | 고정일(양력) | `d{MMDD}-{이름}-…` |
> | 음력 명절 | `lunar{M-D}-{이름}-…` |
> | 그 밖(고유명) | `{kind}-{season}-{이름}-…` |
>
> **② 슬롯 배정** — 대안·조건부 줄을 걷어내면 core 독서가 예외 없이 3칸 또는 4칸이다(4칸 = 제1·시편·제2·복음, 3칸 = 제1·시편·복음). 다만 위치로 우기지 않는다 — **마지막 칸이 복음서가 아니면 복음이 없는 날**로 보고 `second` 로 두고 리포트에 플래그한다(12.31. 예수 이름 전야 1건).
>
> **③ 트랙은 연중 주일에만.** 제목 말미의 `1`·`2` 를 무조건 `reading_track` 으로 읽으면 안 된다 — RCL 트랙은 연중 주일(146건)뿐이고, **재의 수요일·성 금요일·부활대축일의 '2' 는 그날의 다른 독서 선택지**(26건)다. 트랙으로 실으면 있지도 않은 축이 생기고 앱이 "독서 2" 라고 잘못 이름 붙인다 → 대안 세트(`set_no`)로 옮긴다.
>
> **④ 세트 번호는 결정적이어야 한다.** `set_no` 를 기사 배열 순서로 매기면 재스크랩 한 번에 id→독서 매핑이 뒤집힌다(§10 "잠정 순번 id 금지"의 재발). 좌표 밖의 결정적 키로 정렬한다 — **옛 연도판(`2019.9.29.`)은 뒤로**(정본이 언제나 `s1`), 인쇄된 세트 번호 순, 본 독서가 대안보다 앞. 옛 연도판을 버리지는 않는다(ADR-036 §7 밀린 독서 보존).
>
> **⑤ 같은 좌표에 여러 기사** — 독서가 같으면 중복 기사다(`부활4주`/`부활4주일`) → 하나로 합치고 `_sources` 에 둘 다 남긴다. 다르면 그날의 또 다른 세트다 → 세트 번호를 이어 붙여 보존한다.
>
> **⑥ 송가는 기도서 번역이 정본이다** — ADR 원안(`bookId:"lps"` + 부록 장 151+ + `scripture_ref` 병기)을 **개정**한다. 송가는 성무일과 송가 책 **`cant`** 에 손편집 정본으로 들어 있고(데이터 저장소 ADR-039 v2, 13편), 전례에서 실제로 낭송하는 본문이 그 번역이므로 시편 슬롯도 그것을 렌더한다. 기도서에 없는 송가만 공동번역 성구로 폴백한다. 성구 표기로 적힌 범위(`이사 12:2-6`·`루가 1:46하-55`)는 절 번호 체계가 달라 그대로 못 쓰므로 **송가 전체를 낭송**하고 인쇄본이 가리킨 성구는 `scripture_ref` 로 남긴다.
>
> **⑦ 원본 사이트 결함 2건**(근거는 html 캐시) — `에스 4:17(12-16), 17(23-25)` 는 위첨자가 태그와 함께 뭉개져 `4:1712-16` 이 돼 있었고, `이사 55:11-5` 는 사이트 자체의 오타(연중18주 가해 트랙2 = `이사 55:1-5`)였다. 교정 근거는 `readings-report.txt` 에 남긴다.
>
> **⑧ 에스델 하위 절은 아직 적을 수 없다** — `에스 4:17(12-16)` 은 그리스어 에스델 추가 본문의 하위 절(우리 데이터의 `alt_ref` 12-16)을 가리키는데 `verseSpec` 문법에 하위 절 표기가 없다. 지금은 17절 전체를 싣고 label 에 인쇄본 표기를 보존한다. **문법 확장 여부는 후속 결정**(ADR-010 verseSpec 을 건드리므로 북마크와 공유).

### 2. 독서 문자열 파서 — `liturgical/parse_readings.py` (신규)

- ADR-036 정규화기(`normalize.py`)의 **제목→좌표 파싱을 `liturgical/coords.py`로 추출**해 본기도·독서 파서가 공유(단일 출처).
- 책 이름 → bookId 는 `book_mappings.json`의 한글 별칭 표 사용. 미커버 별칭(`사무상/하`·`열왕상/하`·`역대하`·`마카상/하` 등)은 **`aliases_ko`에 보강** — 성서 검색 `in:` 토큰도 함께 개선되는 부수 효과.
- **저녁기도·아침기도 기사는 제외**하고 별도 버킷으로 리포트(감사성찬례 아님 — Phase 4 성무일과 원료).
- 산출물: `lectionary/eucharist-readings.json` + `lectionary/search-lectionary.json`(§5) + 검증 리포트(`readings-report.txt` — 미해석 문자열 0건 목표, 모든 ref가 실존 절 범위인지 대사).
- **제목 정규화 전수 매핑 리포트(육안 검토용):** 제목 정규화는 예외 케이스가 많을 수 있으므로, 리포트에 **정규화 title·좌표 ← 원본 기사 제목** 전수 매핑을 전례 순서로 수록해 눈으로 훑으며 오분류를 잡을 수 있게 한다. (`normalize.py`의 collects 리포트에는 TITLE MAP 섹션으로 이미 구현 — 2026-07-06; readings 리포트도 동일 형식.)

> **개정 (2026-07-13, D2):** "모든 ref 가 실존 절 범위인지 대사" 는 **절 단위 실존 검증**이어야 한다. 장의 최대 절만 비교하면 **장 중간의 결번을 놓친다** — 실제로 그 구멍으로 두 건이 빠져나가 리포트가 "오류 0" 을 찍고 있었다.
>
> - **루가 1:68 이 성경 데이터에 없었다** — `source/luke.md` 에서 `[68]` 마커가 유실돼 즈가리야의 노래 첫 소절이 67절에 흡수돼 있었다. 본문은 화면에 나오지만 **그 절을 번호로 부를 수 없어** 전례독서·인용·북마크·딥링크가 전부 못 찾는 상태였다.
> - **히브 1:12** 도 같은 유실(성탄 낮 제2독서의 마지막 절).
>
> 둘 다 복구했고, 데이터 저장소에 `tests/test_verse_gaps.py` 를 신설해 73권 전체의 결번을 못박았다(남은 결번 10곳은 인쇄본 대조 대기). **교훈: 범위의 끝만 검사하는 검증은 가운데가 뚫린 것을 통과시킨다.**

### 3. 전례시편 — 별도 책 `lps` (계응 구조, 새 마크다운 DSL)

감사성찬례용 시편은 공동번역 시편(`ps`)과 **다른 번역본**이고 **계응(선창◯/응답) 구조**를 가진다. 몇 년 전 개정본이 나왔고 스크랩본은 개정 전 본문이라, **손편집 가능한 마크다운이 정본**이어야 한다.

- 마크다운 DSL·파서(`src/liturgical_psalter.py`)·산출 스키마의 권위 출처는 **데이터 저장소 ADR-039**(D1, 2026-07-09 신설). 요점만: `liturgical/psalter/{n}.md` 편당 1파일, `[N]` 절 / **반행 = 한 줄, 계응 경계 = 개행**(홀수 줄 = 선창, 짝수 줄 = 응답 — 렌더 시 "마지막 줄 아닌 홀수 줄" 끝에 ◯ 표시) / `[-]` = 무번호 후속 구절(원문 `¶`, 렌더 시 ¶ 표시) / 빈 줄 = 연 / `## (N)` = 부(部) — 긴 시편의 봉독 토막 구분, 부의 모든 절에 `section: N`, 렌더 시 인쇄본 그대로 "(N)" 소제목(ADR-039). **입력 문법은 전부 기본 자판 문자** — `◯`·`¶` 같은 전례 글리프는 저장하지 않고 렌더링할 때만 표시한다(자판 입력이 어려움). **표시용 줄바꿈도 저장하지 않는다**(의미 구조만). 파서는 절 블록 줄 수가 홀수면 리포트에 플래그. 템플릿·실전 예 = `liturgical/psalter/TEMPLATE.md`.
- 산출 `bible/lps-{n}.json`은 **기존 장 JSON 스키마와 동형** + additive 확장(`response: true` 세그먼트 = 응답 반행, `versicle: true` 절 = 무번호 구절, `section: N` 절 = 부(部) 번호) → `appendVerses`가 그대로 소화.
- **영광송은 파일에 저장하지 않고 앱 상수로 부착**(150회 반복 제거·단일 출처).
- `books.json`에 `{"id":"lps", "division":"liturgical", …}` append — **division 필터만으로 숨김이 성립**(책 목록 탭·검색 인덱서 모두 division 기반, 확인됨). 송가는 lps 부록 장(151+).
- 최초 생성은 1회성 `liturgical/extract_psalter.py`(html 캐시에서 추출·중복 대사, 스크랩에 없는 편은 갭 리포트). 이후 md 손편집이 정본.

### 4. 표 데이터 — 연중 주간 구간표 · 음력

- **`ordinal-weeks`**: 연중 주일·주간이 어느 날짜 사이에 오는지는 성공회 기도서의 구간표를 따른다(ADR-036 §11의 "끝에서 역산" 난점을 표 조회로 대체). **원본은 사용자가 타이핑하는 간단한 텍스트** `liturgical/ordinal-weeks.txt`(`6주: 2/11-2/17, 5/24-5/30` — 구간 2개는 쉼표), 파싱 스크립트가 `lectionary/ordinal-weeks.json`(`{week, windows:[{from,to},…]}`)으로 변환. **한 주간이 구간을 최대 2개** 가질 수 있다(공현 후 연중 / 성령강림 후 연중). 엔진은 날짜가 연중 절기 스팬 안일 때만 구간을 조회한다.

> **개정 (2026-07-13, D3 구현):** 표 입력 완료(34주 · 구간 38개, 데이터 저장소 `feat/ordinal-weeks`). 구현하며 **윤년에 구간을 날짜 그대로 조회하면 공현 후 구간이 깨진다**는 것이 드러나, 구간마다 조회 기준점을 명시하기로 한다 — JSON 스키마가 `{week, windows:[{from, to, anchor}]}`로 넓어진다(`anchor: "doy" | "date"`).
>
> 표의 구간은 1/7부터 7일씩 끊은 블록이라 **2월이 28일일 때만 맞아떨어진다.** 윤년에는 2/29가 3월 날짜를 하루 밀어 8주 구간(2/25–3/3)이 8일이 되고, 2052년(부활절 4/21)에는 2/25와 3/3이 **둘 다** 그 구간에 들어간다. 3/3은 세례주일부터 세면 아홉 번째 주일이므로 **9주**가 맞고, 9주 구간(3/4–3/7)은 그해 빈 채로 남는다.
>
> | 구간 | `anchor` | 조회 방식 | 같은 뜻 |
> | --- | --- | --- | --- |
> | 공현 후 (1~9주) | `doy` | 월/일이 아니라 **연중 일수**(2/29를 셈)로 맞춘다 | 세례주일(1주)부터 앞으로 센다 |
> | 성령강림 후 (6~34주) | `date` | **월/일 그대로** 맞춘다 | 대림 직전 주일(34주)부터 뒤로 센다 |
>
> 두 기준이 갈리는 이유는 **각 구간이 매달린 기준점이 다르기** 때문이다. 앞 구간은 연초(1/7)에, 뒤 구간은 연말(대림 1주일 = 12/25 직전 네 번째 주일)에 매달려 있고, 그 사이에 2/29가 끼어 있다. 뒤 구간은 7일짜리 날짜 블록이 항상 주일을 정확히 하나씩 품으므로 날짜 조회만으로 윤년에 안전하다.
>
> **검증:** 1900~2100년 전 연도의 연중 주일을 computus로 생성해 표와 대조 — 모든 주일이 정확히 한 주간에 속하고, 공현 후는 1주부터 연속, 성령강림 후는 34주로 끝난다(`tests/test_ordinal_weeks.py` 10건). 표의 두 상한(사순 직전 3/7, 대림 직전 11/20–11/26)이 computus 상한과 일치함도 함께 확인했다. 2052년 윤년 사례는 회귀 테스트로 못박았다.
- **`kasi-lunar.json`**: 설(음 1/1)·추석(음 8/15)의 양력 날짜 lookup, 2025–2050, KASI 기준(ADR-036 §9).

### 5. 전례독서 검색 인덱스 — `lectionary/search-lectionary.json`

`parse_readings.py`가 함께 산출. 워커가 400KB 전체 readings를 파싱하는 대신 **~80KB 수치 인덱스**로 순수 비교만 한다:

```json
{ "names": [ { "id": "advent-3-sun-A", "title": "대림3주", "aliases": [], "year": "A", "label": "대림3주 (가해)" } ],
  "cov":   [ [entryIdx, bookIdx, chapter, vStart, vEnd] ],
  "books": ["gen", "…"] }
```

- 장절 질의("창세 1:3") → `cov` 스캔으로 포함하는 교회력일 목록. 이름 질의("부활") → `names` 매치(별칭 포함 — ADR-036 §2 `aliases`).
- **트랙 라벨 병기:** `title`은 트랙 표기를 제거한 정규화 이름(ADR-036 §2)이므로, `reading_track`이 비-null인 엔트리의 `label`에는 `독서 N`을 병기한다(예: `연중10주 (가해, 독서 2)`) — 트랙 1·2가 같은 이름으로 뭉쳐 구분 불가능해지는 것을 방지. 표시 자리별 규칙은 ADR-038 §표시 라벨.

> **개정 (2026-07-13, D2):** `cov` 는 **연속 구간마다 한 행**을 낸다. ref 하나를 `[min, max]` 한 행으로 뭉개면 끊긴 구간의 빈 절이 딸려 들어가 **그날 읽지도 않는 절이 그날의 독서로 검색된다** — `시편 80:1-7,17-19` 가 `80:1-19` 가 되어 8~16절이 대림1주(나해)로 잡혔다(3,214행 중 465행). 구간별로 쪼개면 3,743행으로 16% 늘 뿐이고 스키마·질의 코드는 그대로다.
>
> `names[].label` 에는 **대안 세트 표식도 병기**한다(`set_total > 1` 이면 `set_note` 또는 `대안 N`) — 트랙만 병기하면 같은 이름의 행이 둘씩 떠 구분되지 않는다(64건).

### 6. 교회력 계산 엔진 — `js/app/liturgical-engine.js` (신규, leaf)

순수 로직은 `// ── BEGIN LITURGICAL_CORE ──` 블록(ADR-013 vm 하네스 유닛 대상). DOM 없음.

- **계산 계층**: `easterDate(y)` = Anonymous Gregorian computus(Meeus/Jones/Butcher — ADR-036 §11 채택안), 파생 이동일 오프셋표(`E−46` 재의 수요일 … `E+56` 삼위일체), 대림1주일 = 12.25 직전 4번째 주일, 주일 주기 A/B/C(대림 시작 기준 3년) · 평일 I/II(홀·짝 달력년), 연중 주차 = ordinal-weeks 구간 조회(절기 우선 — 절기 스팬이 먼저 확정되고 연중 스팬에서만 표 사용, 구간의 `anchor`가 조회 기준을 정한다 — §4 개정 2026-07-13), 설·추석 = kasi-lunar lookup. 날짜 연산은 로컬 `new Date(y,m,d)` + `"YYYY-MM-DD"` 문자열(UTC 오프셋 함정 회피).
- **조회 계층**: `resolveDate(dateStr)` → `{coord, candidates:[관측일…]}`. temporal 좌표 + sanctoral `date` 매치 + `lunar` 매치 + 고유명 특수일(성주간·재의 수요일 후 X요일 등 앵커 파생). **후보 관측일 집합 전체 보존**(ADR-036 §7 밀린 독서 보존) — 공식 1건 선택은 뷰의 몫.
- **후보 표시 순서**: ① 교회력에 따른 축일/재일 독서 → ② 이동 축일 독서 → ③ 나머지(연중 평일/연속) 독서. 대축일·주요축일·사계재(춘계재/하계재/추계재/동계재)에 연중 주간 독서가 전례상 사용되지 않더라도 **후보는 모두 제공**한다 — 독서자의 목적에 맞는 선택은 사용자 몫.
- `findReadings(coord)`/`findCollects(coord)` — ADR-036 §2·§4 폴백(weekday/year `null`=공통·배열=포함, 더 구체적인 레코드 우선). readings는 `reading_track`·`set_no` 별 그룹 반환.
- 데이터 로드: `/data/lectionary/*.json` lazy fetch + 모듈 캐시(`data-fetch.js` 패턴).
- **품계·이동(transfer) 엔진은 이번 범위에서 제외** — 분류 필드(rank·precedence·color)가 ADR-036 2차 보류(전부 null)라 입력이 없다. 입력 없이 규칙만 구현하면 죽은 코드 + 검증 불가. 분류 필드 2차 채움과 함께 후속 PR(ADR-036 미결9와 동기).

## 검토한 대안

### 독서 데이터 정본 형태
- **A. 전용 좌표 스키마 (채택)** — 교회력 좌표·시편 대체본·본기도 조인·검색 축을 한 스키마에서 해결. 렌더는 verseSpec 재사용으로 ADR-035 경로 공유.
- **B. 북마크 export 파일 형식** — 기존 import·읽기 뷰를 그대로 쓸 수 있으나 교회력 좌표가 폴더명 문자열에 실려 검색·캘린더 링크·엔진 조회가 전부 문자열 파싱에 의존. 기각.

### 전례시편 저장
- **A. 별도 책 lps + 새 DSL (채택)** — 독서가 일반 성구처럼 `{bookId, chapter, verseSpec}`로 참조, 마크다운 손편집 = 개정본 교체 경로. 계응 구조는 기존 ADR-006 운문 형식과 달라(절 안 선창/응답 쌍 + 무번호 구절) 별도 파서가 안전(기존 parser.py 불변, 회귀 위험 0).
- **B. 독서 세트에 인라인 본문** — 같은 시편이 여러 교회력일에 중복 수록, 개정 시 전체 재생성. 기각.

### 연중 주차 산정
- **A. 기도서 구간표 조회 (채택)** — 권위 출처(기도서)와 1:1, 데이터로 검증 가능.
- **B. 왕이신 그리스도(연중34주)에서 역산** — ADR-036 §11 초안. 구현 가능하나 기도서 표와 어긋날 위험을 코드가 지게 됨. 표가 있으므로 기각.

### 검색 인덱스
- **A. 수치 cov 역인덱스 + names (채택)** — 워커에 verseSpec 파서 복제 불필요, 소용량.
- **B. 워커가 readings 전체 로드** — 파싱 로직 중복·용량 5배. 기각.

## 미결 사항

1. **송영 A vs B 구분 규정** — 성부 호칭 송영 두 형(A·B) 중 B를 언제 쓰는지 성공회 기도서에서 확인(서울교구 실사용 데이터엔 B가 0건 — ADR-036 §3 개정 2026-07-06의 상수 표·실증 참조). `ending` 배정은 호칭 기준 A(성부)/C(성자)로 시작.
2. ~~**ordinal-weeks 표 입력** — 사용자가 기도서 보고 타이핑(D3).~~ **해소 (2026-07-13)** — 34주 · 구간 38개 입력 완료, 파서·검증까지 D3 구현(§4 개정). 표가 computus와 어긋나지 않음을 1900~2100년 대조로 확인했다.
3. **조건부 세트 3건**(주의 변모 주일로 지키는 경우)의 캡션 표현 검토(ADR-038 `set_note`). — D2 에서 조건 원문을 `set_note` 에 보존했다. 남은 것은 **표시 문구**.
4. ~~스크랩에 없는 시편 편(갭 리포트 결과에 따라) — 사용자 타이핑 보충 여부.~~ **해소 (2026-07-13)** — 전례시편 150편 전편 입력 완료(D1).
5. **교회력 격자의 빈 칸 6개** (사순5주 나해 · 연중5·8주 나해 · 연중9주 가·다해 · 연중10주 다해) — **원료에 그 날 기사가 없다**. 스크랩 재수집으로 채울지, 기도서를 보고 입력할지 결정 필요(`readings-report.txt` 에 노출).
6. **에스델 하위 절 표기** — `verseSpec` 문법 확장 여부(§1 개정 ⑧). ADR-010(북마크)과 공유하는 문법이라 함께 결정한다.
7. **날짜를 모르는 성인일 3건**(성 십자가 발견 · 사도 성 바르톨로메오) — 좌표에 `date` 가 없어 엔진이 날짜로 찾을 수 없다(ADR-036 §9 성인 달력 2차와 동기).
