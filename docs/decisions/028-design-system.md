# ADR-028: 디자인 시스템 — 토큰 사다리 + HIG 정렬

- 일시: 2026-06-02
- 상태: 승인됨 — 구현 완료 (2026-06-02)
- 관련 ADR: ADR-023(토글 스위치), ADR-024(책 목록 탭·헤더 내비), ADR-025(스크롤 elevation), ADR-022·027(인용·주석)
- 권위 문서: 루트 [`DESIGN.md`](../../DESIGN.md)

## 맥락

`css/style.css`(4,800줄+)는 **색상만** CSS 변수로 토큰화되어 있었다 — `:root` /
`[data-theme="dark"]` / `[data-color-scheme]` 3중 구조로 잘 정리된 팔레트. 그러나 그
밖의 모든 시각 속성은 즉흥적(ad-hoc)이었다:

- **타이포 스케일 부재** — `0.65rem`~`1.6rem` 사이 14개 이상의 폰트 크기가 일관 기준 없이
  난립.
- **간격 스케일 부재** — `0.15rem`~`4rem` 사이 20개 이상의 rem 값이 문맥별로 제각각.
- **elevation 부재** — 그림자가 `0.08 → 0.12 → 0.18 → 0.25` 라는 **암묵적** 단계로만
  존재하고 토큰화되지 않음. 다크 모드 보강도 컴포넌트마다 손으로 처리.
- **컨트롤/터치타깃 불일치** — `28px·32px·44px·2.2rem·2.4rem` 혼재. 일부 헤더 아이콘
  버튼은 기본 폰트에서 ≈40px 로 **Apple HIG 44pt 최소 탭 영역 미달**.

새 UI 작업마다 "이 간격이 맞나, 이 폰트 크기가 다른 데랑 같은가" 를 매번 추정해야 했고,
일관성이 작성자 기억에 의존했다. 단일 권위 출처가 필요했다.

방향은 사용자와 합의했다: (1) 시각 회귀를 최소화하는 **토큰 레이어 + 점진 치환**(전면
재작성 아님), (2) 성경 **본문은 Serif·넉넉한 행간 유지**하고 chrome UI 에만 Apple HIG
적용, (3) 간격은 **px 기반 8pt 그리드**(HIG 정합, 폰트 크기 설정과 독립).

## 결정

### 1. `:root` 에 토큰 사다리 추가 — 순수 추가, 하위호환 유지

기존 토큰명(`--bg`…, `--radius`, `--shadow`, `--max-width`)은 **그대로 둔다**.
신규 토큰만 추가:

- **반경** `--radius-xs(4) /sm(6) /(8) /md(10) /lg(12) /modal(14) /xl(16) /pill(999)`.
  기존 코드에서 실제 쓰이던 값에 매핑하므로 치환 시 회귀 0.
- **elevation** `--shadow-1(=--shadow) /2 /3 /sheet /4`. 라이트 값은 기존 암묵 단계를
  명문화한 것이고, **다크는 `[data-theme="dark"]` 에서 같은 토큰을 더 무거운 alpha 로
  자동 치환**한다 — ADR-025 가 헤더에만 손으로 하던 다크 보강을 전 표면으로 일반화.
- **간격(8pt px)** `--space-0_5(2) …/4(16) /6(24) /8(32) /10(40) /12(48)`.
- **타이포(rem)** `--font-2xs(0.7) … /2xl(1.6)`, `--leading-tight … loose`.
  rem 으로 둔 이유: 설정의 글자 크기가 루트 폰트를 바꾸면 타이포가 함께 스케일해야 하기
  때문(Dynamic Type).
- **컨트롤** `--touch-target(44) · --control-h(36) · --control-h-sm(28) · --icon-btn(40)`.
- **모션** `--duration-fast(0.18) /base(0.25) /slow(0.4) · --ease-out`.
- **내비(예약)** `--tabbar-h(49) · --tabbar-icon(24) · --sidebar-w(260) · --nav-z(30)`
  — §3 의 후속 구현이 사용.

### 2. 간격은 px, 타이포는 rem

HIG의 8pt 그리드를 따르되 **레이아웃(간격)과 콘텐츠(폰트)를 분리**한다. 결과적으로
사용자가 글자 크기를 키워도 패딩·여백은 고정된다 — 이는 HIG 상 올바른 동작(레이아웃은
타이포가 아니라 디바이스/캔버스에 종속). 본문 가독성에 영향이 없도록 본문 영역 패딩은
보수적으로만 치환한다.

### 3. 점진 치환 — 대표 컴포넌트 우선, 값 동일

전면 재작성이 아니라 가시성 높은 지점만 토큰 참조로 교체한다. 토큰 값이 기존 값과
**정확히 같은** 곳부터(회귀 0): 헤더 스크롤 그림자(`--shadow-3`), 모달
(`--radius-modal`·`--shadow-4`), 바텀 시트(`--radius-xl`·`--shadow-sheet`), 팝오버
(`--radius-lg`). 나머지 ad-hoc 값은 그대로 두되 `DESIGN.md` 가 정규 스케일을 제시하므로
후속 작업이 점진적으로 수렴한다.

### 4. HIG 44pt 터치타깃 — 헤더 아이콘 버튼 보강

기본 폰트에서 ≈40px 였던 헤더의 뒤로/북마크 아이콘 버튼을 `2.45rem`(≈44px @18px root)
으로 올려 HIG 최소 탭 영역을 충족하면서 rem 스케일은 유지한다. 고정 크기 컨트롤(오디오
재생 버튼)은 `min-width/min-height: var(--touch-target)` 로 바닥을 깐다.

### 5. 의도적 HIG 이탈을 명문화

- 본문은 `Noto Serif KR` + line-height 1.8 유지(성경 가독성·전례 전통). HIG 본문
  산세리프 대신.
- **frosted glass 는 하단/오버레이 표면에 적용**(오디오 바·절 선택 바·검색 스크림의
  `backdrop-filter: blur`). **헤더만** 솔리드로 두고 스크롤 elevation(`--shadow-3`)으로
  깊이를 표현한다 — iOS toolbar tinting / PWA status bar 비동기 문제 회피(ADR-025). 즉
  "frosted glass 전역 미적용" 이 아니라 iOS HIG 의 상단/하단 차등 처리를 따른 것.

`DESIGN.md` §11 에 적용/이탈 표로 명시.

### 6. 테마색 적용 범위 한정 (개정 2026-06-03)

> **개정 (2026-06-03):** 색상 스킴(`[data-color-scheme]`)이 **chrome 전반**(버튼·링크·
> 포커스 링·토글·활성 상태 등 `--accent` 사용처 ~100곳)을 함께 물들이던 동작을 거둔다.
> 테마색은 **절 번호·단락 기호·FAB·오디오 플레이어 네 곳에만** 칠해지도록 한정한다.
>
> 토큰 구조를 둘로 가른다:
> - **`--theme`** (신규) — 스킴을 추종하는 유일한 강조색. 진한 채도로 통일하며 위 네 곳만 참조.
>   기존 `--accent` 의 스킴별 값을 그대로 물려받는다(네이비/빨강/초록/보라 × 라이트/다크).
> - **`--accent` / `--accent-light`** — 이제 **고정 중립 차콜/회색**(라이트 `#3a3a42` /
>   다크 `#b8b8c2`). `[data-color-scheme]` 블록에서 더 이상 오버라이드하지 않으므로 어떤
>   테마색을 골라도 chrome 톤은 불변. 라이트/다크 전환만 따른다.
> - 기존 `--verse-num` · `--paragraph-mark`(스킴별 muted 변형) 토큰은 제거하고 `--theme`
>   로 통일 — 사용자 결정 "진한 채도로 통일". 절 번호·단락 기호가 기존보다 또렷해진다.
>
> 의도: 테마색을 chrome 전반을 칠하는 도구에서 **읽기 화면의 절제된 시그니처 강조**로
> 좁힌다. chrome 은 브랜드 없는 중립 톤으로 일관되게 두어 스킴 선택이 본문 읽기 경험에만
> 영향을 주도록 한다. 구현은 `--accent` 사용처를 건드리지 않고(자동 동결) 네 곳만
> `--theme` 로 재지정해 회귀를 최소화했다. DESIGN.md §2 갱신.
>
> **다크 대비 보정:** `--accent` 가 다크에서 밝은 회색(`#b8b8c2`)이 되면서, accent 채움 위에
> `color: #fff` 를 쓰던 chrome 컨트롤(toolbar·팝오버·장 격자·재개 배너·검색 페이지네이션 등
> ~17곳)은 흰 글자 대비가 무너진다. 이를 코드베이스에 이미 우세하던 관용구
> **`color: var(--bg)`** 로 통일했다 — 라이트=크림 / 다크=네이비로 자동 뒤집혀 양쪽 AA 확보.
> 빨강 파괴 동작 버튼(삭제·캐시 비우기)만 `#fff` 유지. (Bugbot 리뷰 PR #180 반영)

### 7. 토큰 전면 채택 — 스케일 스냅 (개정 2026-06-03)

> **개정 (2026-06-03):** §3(점진 치환)은 "값이 정확히 같은 곳만" 우선 치환이었으나,
> 후속 UI 리팩토링에서 **ad-hoc 값을 가장 가까운 토큰으로 전면 스냅**해 `css/style.css`
> 전반이 토큰을 따르도록 수렴시켰다(사용자 합의 "스케일 스냅"). 토큰과 정확히 일치하지
> 않던 값은 미세하게 조정됨(시각 회귀 최소).
>
> - **타이포** — font-size 108곳 → `--font-*`, line-height → `--leading-*`.
> - **간격** — margin/padding/gap 193곳 → `--space-*` 8pt(px). calc/env/positioning·
>   그리드 초과·기능적 예약(3rem·6rem 등)·읽기 칼럼 패딩은 보수적 제외.
> - **반경** — 34곳 → `--radius-*`. 마이크로(2px 그래버·1px 진행바)는 유지.
> - **elevation** — 토큰과 정확 일치분만 `--shadow-*` (나머지는 포커스 링·다층 커스텀).
> - **모션** — 토큰을 실사용에 맞게 조정 후 스냅: `--duration-fast` 0.18→**0.15s**,
>   신규 `--duration-instant(0.1)`·`--ease-standard(cubic-bezier(0.4,0,0.2,1))`. 듀레이션
>   64줄 스냅. CSS 기본 `ease`·스피너·1회성 곡선은 유지.
> - **신규 토큰** — `--icon-btn-touch(2.45rem)`: rem 스케일 헤더 아이콘 박스(≈44px @18px,
>   Dynamic Type 추종). `.title-back-btn`·`.title-bookmark-btn`·`#cite-sheet-back`·
>   `#cite-sheet-close` 공유. 고정 px `--touch-target` 과 역할 구분.
>
> 부수 버그 픽스: 인용 시트 헤더 높이가 back 버튼 토글로 변하던 문제를 닫기 버튼을
> `--icon-btn-touch` 로 통일해 해소(HIG 44px 터치타깃도 충족). DESIGN.md §3·4·5·8·9 갱신.

## 적응형 내비게이션 (결정됨 — 구현 대기)

향후 화면 내비를 HIG 적응형 패턴으로 재편한다 — **모바일(≤768px) 하단 탭 바**(좌→우:
홈·검색·북마크·노트·설정, 5탭, `--tabbar-h` 49px, safe-area, 활성=`--accent`) +
**데스크탑(≥769px) 사이드바**(`--sidebar-w` 260px). `노트` 는 미구현 신규 기능 슬롯.
현행 상단 헤더의 아이콘 버튼·검색 FAB 가 이 목적지들로 이전된다.

이번 ADR 에서는 **레이아웃 토큰 예약 + `DESIGN.md` §7 스펙 문서화**까지만 한다. 실제
탭바/사이드바 렌더·라우팅 재편·노트 기능은 별도 ADR/PR 로 진행한다. 그래서 이번 작업의
헤더 정리는 44pt 터치타깃·토큰화 같은 무난한 수준에 한정하고 구조는 건드리지 않는다(곧
대체될 영역에 과투자 방지).

## 검토한 대안

- **전면 리디자인(HIG 정석)** — 타이포·간격·컴포넌트 비주얼을 적극 재설계. 임팩트는 크나
  4,800줄 CSS 의 시각 회귀 위험이 높고 ADR 다수 개정 필요. 토큰 레이어 + 점진 치환으로
  같은 일관성을 낮은 위험에 확보 가능하다고 판단해 보류.
- **문서화만(코드 무변경)** — 권장 스케일을 `DESIGN.md` 에만 적으면 코드와 문서가
  즉시 괴리. 토큰을 `:root` 에 실제로 심어 코드가 따르게 했다.
- **본문까지 시스템 산세리프 통일** — iOS 네이티브 느낌은 강해지나 성경 본문의 전통적
  가독성이 약화. 본문 Serif 유지로 결정.
- **rem 기반 간격 유지** — 폰트 크기 설정에 간격도 함께 스케일. 그러나 8pt 그리드와
  어긋나고, 레이아웃이 콘텐츠 크기에 종속되는 비-HIG 동작. px 8pt 로 분리.

## 영향

- `css/style.css` — `:root`(+다크)에 토큰 블록 추가, 대표 컴포넌트 점진 치환. 토큰 추가는
  값 무변경이라 시각 회귀 없음. 다크 모달/시트 그림자는 elevation 일반화로 약간 더 짙어짐
  (의도된 개선).
- 신규 `DESIGN.md`(루트) — 디자인 단일 권위 출처.
- `docs/architecture.md` 부록 A · `CLAUDE.md` 현재 상태 갱신.
- 유닛 테스트·tsc 영향 없음(CSS 무관). DOM 구조 무변경이라 e2e 영향 적음.
