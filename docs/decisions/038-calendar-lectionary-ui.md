# ADR-038: 교회력 캘린더 · 전례독서 뷰 · 검색 이원화 UI

- 일시: 2026-07-04
- 상태: **승인됨 — 구현 대기** (구현 PR 시리즈 A2~A4)
- 관련: ADR-037(전례독서 데이터·엔진 — 본 UI의 입력), ADR-036(본기도·좌표), ADR-035(북마크 읽기 뷰 — 렌더 패턴 원형), ADR-029/030/031(탭 바·모핑·탭 히스토리), ADR-032(오버레이·뷰 레이어), ADR-033(검색 옵션), ADR-034(라우팅 분할)

## 배경

ADR-037의 데이터·엔진 위에 얹는 세 UI: ① 교회력 캘린더 탭, ② 전례독서 본문 뷰, ③ 검색 이원화(성서 본문 / 전례독서). 모두 기존 확장 지점(라우팅 `parsePath`/`route`, 탭바 `syncTabBarActive`, `appendVerses` 렌더, 검색 워커 메시지)을 따른다.

## 결정

### 1. 캘린더 탭 — 홈 옆 두 번째 위치

`index.html` `#tab-bar`에 `<a data-tab="calendar" href="/calendar">`를 **홈 바로 다음(두 번째)**에 추가. 노트 탭(비활성 숨김)은 보존 — 나중에 부활 예정. `tabbar.js syncTabBarActive`의 세그먼트 매핑에 `calendar` 추가(폭·인디케이터는 개수 기반 자동). 아이콘은 기존 스트로크 스타일(광학 크기 정규화 규칙 준수) 캘린더 글리프.

### 2. 캘린더 뷰 — 월간·주간·목록 3뷰 (`js/app/calendar.js` 신규)

- **라우트**: `/calendar`(월간, 이번 달) · `/calendar/YYYY-MM`(월간) · `/calendar/w/YYYY-MM-DD`(주간) · `/calendar/list`(목록 — 오늘부터 교회력일 아젠다). `routing.js parsePath()` 분기 + `route()` 디스패치(ADR-034 관례).
- 상단에 **뷰 전환 세그먼트**(월간|주간|목록). 이전/다음 이동 + "오늘" 버튼.
- 순수 함수 `buildMonthGrid`·`buildWeekStrip`·`buildAgendaList`(ADR-013 유닛 대상) — 엔진 `resolveDate`를 주입받아 셀/행 데이터를 만든다.
- 셀/행 = 날짜 + 대표 교회력일명(고유명 우선, 격자 좌표는 주일만 `대림1주` 표기) + 절기색 점. **색은 MVP에서 season 파생색** — ADR-036 §8 명시 `color`가 2차에서 채워지면 교체(뷰 주석으로 한계 명시).
- 셀/행 탭 → `/lectionary/YYYY-MM-DD`. 데스크톱도 동일 전체화면 뷰(ADR-035 `/read` 선례 — 드로어 폴백 없음).

### 3. 전례독서 본문 뷰 (`js/app/lectionary.js` 신규)

- **라우트 — 날짜 기반이 정본**: `/lectionary`(오늘) · `/lectionary/YYYY-MM-DD`. 상태는 쿼리 `?obs=&track=&set=&collect=`. **보조 라우트** `/lectionary/d/<slug>`(ADR-037 안정 슬러그) — 날짜 없는 검색 결과의 착지점. 근거: 날짜 URL은 공유 즉시 의미가 확정되고 엔진이 좌표를 재계산하므로 데이터 개정에도 안정.
- **탭 분류: 캘린더 탭 소속** — 캘린더에서 날짜를 골라 들어가는 하위 화면이므로 캘린더 컨텍스트 유지(ADR-035의 `/read`=홈 선례와 다른 선택 — 봉독 자체가 목적인 `/read`와 달리 이 뷰는 날짜 탐색의 하위).
- **본문 순서: 본기도 → 제1독서 → 시편 → 제2독서 → 복음.**
- **본기도 블록 — 두 축을 구분한다**:
  - 같은 관측일의 **자유선택 대체안**(`collect_no`) → **세그먼트 탭**으로 택1(`?collect=N`). 탭 패턴은 `views.js buildDivisionTabs` 슬라이드 인디케이터 재사용.
  - 서로 다른 관측일이 겹쳐 **따라오는 본기도**(당일 + 성인축일 + 교회 축성일/축복일) → **순서대로 열거**.
  - **송영은 앱 상수 3종** 중 하나를 **열거 마지막에 1회만** 부착(ADR-036 §3 개정 — 선택 원칙은 기도서 확인 후. 시편 끝의 영광송과 별개 용어).
- **독서 렌더 — ADR-035 경로 재사용**: 필요한 장 JSON 전부 선로드(중복 제거) → stale 체크 → 동기 DOM 커밋(`bookmark-read.js` 패턴). 독서별 섹션 제목(label) + refs의 장별 `<article>` + `appendVerses(…, {hideCites:true})`(장 경계 병합·절 부분집합). 절 필터 로직(`_specCoversVerse` 등)은 `verse-spec.js`(leaf)로 승격 export해 bookmark-read와 공유(복제 금지).
- **계응 시편 조판**: `lps` 장의 `response` 세그먼트(응답 반행)·`versicle` 절(무번호 ¶)을 `appendVerses`가 additive로 처리 — 응답 반행 시각 구분 + `◯` 표지 + **영광송 상수 부착**. 송가는 시편 장/절 표기 옆에 명칭 병기(예: `루가 1:46하-55, 성모송가`).
- **대안 세트·트랙 전환**: `set_total>1` → "대안 독서" 세그먼트(조건부 세트는 `set_note` 캡션), `reading_track` 1/2 존재 시 "독서 1/독서 2" 토글. **트랙은 전례년 단위로 localStorage 기억**(ADR-036 §2 독서 연속성 — 그해 첫 선택이 기본값, 회별 변경 허용).
- **후보 관측일이 여럿이면**(축일∧평일 겹침 등) 헤더 아래 관측일 스위처 칩(`?obs=`). 표시 순서는 ADR-037 §6 — ① 축일/재일 → ② 이동 축일 → ③ 나머지. **전례상 사용되지 않는 후보(축일에 밀린 연중 독서)도 모두 노출** — 독서자의 목적에 맞는 선택은 사용자 몫(ADR-036 §7 밀린 독서 보존).

### 3-1. 표시 라벨 규칙 — 트랙·주기 병기

데이터의 `title`은 주기·트랙 표기를 뺀 정규화 이름(ADR-036 §2)이다. 따라서 **전례독서 엔트리를 이름으로 나열하는 모든 자리**에서는 표시 계층이 라벨을 조합한다:

- **라벨 = `title` (+ 주기 라벨) (+ `독서 N`)** — `reading_track`이 비-null이면 반드시 트랙을 병기한다. 예: `연중10주 (가해, 독서 1)` / `연중10주 (가해, 독서 2)`. 이 규칙이 없으면 트랙 1·2가 같은 이름으로 표시돼 구분할 수 없다.
- 적용 자리: **캘린더 목록(아젠다) 뷰**에서 하루의 독서 세트를 나열할 때, **검색 결과 카드**, **관측일 스위처 칩**. (월간·주간 셀은 대표 교회력일명만 — 트랙 구분은 셀에 넣지 않고 독서 뷰의 토글이 담당.)
- **독서 뷰 본문 제목에는 트랙을 표시하지 않는다** — 트랙 선택은 상단 "독서 1/독서 2" 토글이 담당하므로 본문 제목까지 반복하지 않는다.
- 검색 결과에서 같은 교회력일의 두 트랙이 모두 매치되면 한 카드로 묶고 매치된 트랙을 표기(예: "독서 1·2 포함"), 한 트랙만 매치되면 그 트랙을 병기(예: "독서 2에 포함") — 질의한 성구가 어느 트랙 독서인지가 유용한 정보다.

### 4. 검색 이원화

- **검색 화면 상단에 탭** — `[성서 본문 | 전례독서]`. URL `?mode=lectionary`(기본 bible) — ADR-031 탭 복원·공유 일관성을 위해 상태를 URL에 인코딩(ADR-033과 동일 원칙). `search.js renderSearchView/renderSearchResults`에 mode 분기; lectionary 모드에서는 책 픽커 대신 안내 placeholder("교회력일 이름 또는 창세 1:1").
- **워커 확장**(`search-worker.js`): `{type:"search", mode:"lectionary", q}` — ① 장절 질의면 기존 `tryVerseRef`(보강된 별칭 포함)로 파싱 후 `search-lectionary.json`의 `cov` 스캔 → 포함하는 교회력일 목록, ② 아니면 `names` 부분 문자열 매치(별칭 포함). 결과 카드 = 교회력일 라벨 + 매치 슬롯 표시 + `/lectionary/d/<slug>` 링크.
- 기존 성서 검색 경로(청크·RLE·`in:` 토큰)는 무변경.

### 5. SW·캐시

`sw.js cacheNameFor`에 `/data/lectionary/` → DATA_CACHE 라우팅 추가. 무효화는 bible-manifest 편입(ADR-037 §7)으로 ADR-021 흐름 자동.

## 검토한 대안

### 캘린더 탭 배치
- **A. 홈 옆 두 번째 신규 탭 (채택, 사용자 결정)** — 노트 탭은 보존(나중 부활). 노트가 숨김 상태라 실표시 4탭 유지.
- B. 노트 탭 자리 대체 — 노트 부활 시 재배치 필요. 기각.

### `/lectionary` 탭 분류
- **A. 캘린더 탭 (채택)** — 날짜 탐색의 하위 화면.
- B. 홈 탭(ADR-035 `/read` 선례) — 봉독 맥락은 같으나 진입·복귀 동선이 캘린더. 기각.

### 독서 뷰 URL
- **A. 날짜 정본 + 슬러그 보조 (채택)** — 공유 안정 + 검색 착지 모두 해결.
- B. 슬러그 단독 — "오늘의 독서" 진입마다 좌표 계산 후 리다이렉트 필요. 기각.

### 검색 이원화 UI
- **A. 검색 화면 상단 탭 (채택, 사용자 결정)** — 한 화면에서 축 전환.
- B. 별도 검색 뷰(/calendar 내 검색) — 검색 동선 분산. 기각.
