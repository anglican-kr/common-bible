### 🟡 [Medium] `_downloadAndMerge` 내 JSON 파싱 미보호

**파일**: `js/drive-sync.js:101`  
**문제**: Drive에서 받은 응답에 대해 `res.json()`을 try-catch 없이 호출하여, 예상치 못한 응답(HTML 유지보수 페이지 등)이 오면 처리되지 않은 Promise rejection 발생.

**코드**:
```js
const res = await _driveRequest(`/files/${fileId}?alt=media`);
if (!res.ok) return;
const remote = await res.json();  // ← try-catch 없음
```

**권고**: `res.json()`을 try-catch로 감싸고 파싱 실패 시 조용히 종료:
```js
let remote;
try { remote = await res.json(); } catch { return; }
```

**상태**: ✅ 수정 완료

### 🟡 [Medium] 401 처리 시 다중 재인증 레이스 컨디션

**파일**: `js/drive-sync.js:43-52`  
**문제**: 동시에 여러 Drive API 요청이 실행 중일 때 각각 401을 수신하면, 각자 `_silentSignIn()`을 호출. 재인증 성공 콜백(`_onTokenResponse`)이 여러 번 실행되어 `_downloadAndMerge()`가 중복 실행, 결과적으로 동시 `_upload()` 요청이 발생해 마지막 요청이 이전 기록을 덮어쓸 수 있음.

**코드**:
```js
if (res.status === 401) {
  _accessToken = null;
  _updateSettingsUI();
  if (localStorage.getItem(SYNC_ENABLED_KEY) === "1") _silentSignIn();
  // ↑ 동시 요청 개수만큼 호출됨
  throw new Error("token expired");
}
```

**권고**: 재인증 진행 중 플래그 추가:
```js
let _isRefreshing = false;

if (res.status === 401) {
  _accessToken = null;
  _updateSettingsUI();
  if (!_isRefreshing && localStorage.getItem(SYNC_ENABLED_KEY) === "1") {
    _isRefreshing = true;
    _silentSignIn();
  }
  throw new Error("token expired");
}

// _onTokenResponse 시작 시 _isRefreshing = false로 리셋
```

**상태**: ✅ 수정 완료 (`_isRefreshing` 플래그 + `_handle401` 헬퍼로 구현)

### 🔵 [Info] GIS 외부 스크립트 SRI(Subresource Integrity) 미적용

**코드**:

**상태**: 수용 (CSP 도메인 제한으로 완화)

### 🔵 [Info] 개발용 OAuth Client ID 프로덕션 번들 포함

**파일**: `js/drive-sync.js:1-3`  
**문제**: `location.hostname === "localhost"` 분기로 두 개의 Client ID가 소스에 하드코딩됨. Client ID는 공개 정보이나, 개발용 Client ID가 프로덕션 배포 파일에 불필요하게 포함됨.

**코드**:
```js
const CLIENT_ID = location.hostname === "localhost"
  ? "359209354241-esbmeba2ku58depo9fgg08v52crfthot.apps.googleusercontent.com"
  : "359209354241-do8kgvtcbnfvrge01f5hj29fee9cg195.apps.googleusercontent.com";
```

**권고**: Google Cloud Console에서 개발용 Client ID의 **Authorized JavaScript origins**가 `http://localhost`로만 제한되어 있는지 확인. Client ID 자체는 비밀이 아니므로 소스 노출 자체는 보안 위협이 아님.

**상태**: 수용 (개발 Client ID의 origin이 localhost로 한정되어 있음을 확인)
