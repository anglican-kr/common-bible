# QA 보고서: 인용·주석 칩 렌더링 유닛 테스트

- 일시: 2026-05-23
- 범위: ADR-022 Phase 2 의 인용 칩(`<cite>`)·주석(`[^id]`) 표시 기능
- 관련 이슈: #135 (Phase 2 — UI 렌더)
- 파일: `tests/unit/citations.test.js` 20 케이스

## 무엇을 확인했나

성경 본문에서 다른 본문을 인용한 부분에 작은 회색 "출처 버튼"을 보여주고,
번역자 주석을 본문 아래에 펼쳐 보이는 기능의 **렌더링 로직**을 자동 검증했다.
실제 화면에 그려지는 모양(테마·크기·색)이 아니라 "어디에 무엇이 어떤
모양의 데이터로 만들어지는가" 를 본다.

## 검증한 영역

### 1. 같은 출처가 연속된 절에 나올 때 한 번만 묶어 표시 (dedup)

예: 마태 4:15-16 이 함께 이사야 8:23-9:1 을 인용. 두 절 각각에 칩이 따로
달리면 같은 출처가 두 번 보이는 시각 노이즈. 검증 결과: 두 절이 같은 출처를
갖고 인접해 있으면 **마지막 절에만** 칩이 그려진다. 사이에 인용 없는 절이
끼면 dedup 이 깨져 양쪽 절 모두 표시 — 의도된 동작.

다음 차이가 있으면 같은 그룹으로 묶지 않는다:
- 출처(`src`) 가 다르면
- 사본 전통(`tradition`, 예: 칠십인역) 이 다르면
- 병행 참조(`parallels`, 예: 신명 5:17) 가 다르면

### 2. 칩 텍스트 조립 형식

| 입력 | 칩 표시 |
|---|---|
| `src` 만 | `(이사 53:5)` |
| `src` + 사본 전통 | `(이사 40:3 · 칠십인역)` |
| `src` + 병행 참조 | `(출애 20:13 · 신명 5:17)` |
| 셋 다 | `(이사 40:3 · 칠십인역 · 마르 1:3)` (사본 전통이 출처 바로 뒤) |
| 다중 병행 | `(출애 20:13 · 신명 5:17 · 마르 12:29)` |

빈 병행 배열은 없는 것과 동등 취급.

### 3. 칩 DOM 구조

- 칩은 `<button>` 으로 만들어진다 (키보드 접근성 — Tab·Enter 동작 가능)
- 산문 인용은 인라인, **운문 인용은 별도 줄** 로 표시(시 행 구조 보존)
- `data-cite-src`·`data-cite-parallels`·`data-cite-tradition` 속성에 원본 값이
  실려 있어, 칩 클릭 시 시트가 어떤 본문을 보여줄지 판단
- 스크린리더용 `aria-label` 에 칩 텍스트 + "본문 보기" 안내가 들어 있다

### 4. 주석 단락 구조

- 절 본문 아래에 `<div class="verse-note">` 단락으로 표시
- `role="note"` 로 보조 정보임을 명시
- "갈릴래아 — 갈릴래아는 당시 로마의 직할령이…" 형식 (anchor 어절 + 본문)

## 무엇을 확인 안 했나 (의도적 범위 제외)

- **칩 클릭 시 바텀 시트 동작** — DOM 마운팅·fetch·focus trap 이라 e2e 영역.
  로컬 수동 확인으로 검증 권장
- **실제 시각 표현** — 색·여백·폰트는 CSS 라 유닛 테스트 영역 밖
- **iOS/Safari 같은 플랫폼 차이** — 실기기 또는 e2e 가 책임
