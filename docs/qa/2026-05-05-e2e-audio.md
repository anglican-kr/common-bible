## 4. 발견된 이슈 / 중요 발견

| # | 위치 | 내용 |
|---|------|------|
| 1 | `showAudioPlayer()` | `audio = new Audio()` 객체가 DOM에 append되지 않아 `document.querySelector('audio')`로 접근 불가. `Audio` constructor를 intercept해 `window.__testAudio`에 참조를 저장하는 mock 사용. |
| 2 | `test_longpress_reveals_mobile_actions` | 전체 suite 실행 시 시스템 부하로 롱프레스 타이밍이 빠듯해 timeout 발생. `_longpress()` wait에 100ms buffer 추가로 수정. |

## 5. 비고

- 오디오 재생에 실제 MP3 파일이 불필요한 mock 방식 채택:
  - `Audio` constructor intercept → `window.__testAudio` 참조
  - `play()/pause()` override → 즉시 이벤트 발화
  - route에서 audio 요청에 빈 body 200 응답
- `timeupdate` 1초 debounce, `ended`·`seek`는 이벤트 직접 dispatch.
