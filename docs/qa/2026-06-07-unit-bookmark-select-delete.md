# 유닛 테스트 보고서: 북마크 선택 모드 (공유·이동·삭제)

**날짜:** 2026-06-07
**범위:** `js/app/bookmark.js` — 북마크를 여러 개 골라 **공유·이동·삭제**하는 "선택 모드"를 화면 안에서 직접 고르는 방식으로 새로 짰다(옛 체크박스 창 제거). 곁들여 행 삭제의 작은 버그도 고쳤다.
**관련 ADR:** ADR-029(모바일 탭 바 — 선택 모드), ADR-010(북마크 — 행 삭제)
**관련 PR:** (작성 중, 브랜치 `feat/bookmark-select-delete`)

## 한 줄 요약

북마크 목록 우측 위 ⋯ 메뉴의 **"선택"**을 누르면, 보고 있던 목록 그대로 위에서 항목을 골라(각 줄 앞 동그란 선택 표시) 화면 아래 막대로 **공유·이동·삭제**할 수 있다(아이폰 사진·메일·파일의 "선택" 모드와 같은 방식). 옛날엔 별도 창이 떠서 거기서 골라 지우는 것만 가능했다.

## 무엇이 바뀌었나 (사용자 입장)

- **진입:** ⋯ 메뉴의 **"선택"**(새 폴더·내보내기·가져오기 바로 아래). 누르면 줄마다 앞에 동그라미(○)가 나타나고, 누르면 ✓로 켜진다. 폴더를 고르면 안의 항목도 함께 표시된다.
- **아래 막대:** **공유 · 이동 · 삭제** 세 버튼이 든 알약(pill) + 오른쪽에 **취소(✕)**. 알약은 내용만큼만 넓고 취소까지 늘어나지 않는다(아이폰과 같은 모양). 위에는 "N개 선택됨" 안내.
- **제목줄:** ⋯/🛈 자리가 **"전체 선택"** 토글로 바뀐다(전체 삭제 = 전체 선택 후 삭제).
- **공유:** 고른 북마크의 **링크**(`https://bible.anglican.kr/…`)를 만들어 기기의 공유 시트로 넘긴다. 공유 시트에서 "복사"를 누르면 **그 링크가 복사**된다(성경 본문이 아니라 링크 — 앱의 본문 복사 기능과는 별개다). 데스크탑 등 공유 기능이 없으면 링크를 클립보드에 복사한다.
- **이동:** 폴더 목록 창(새 폴더… / 최상위 / 폴더들)이 뜨고, 대상을 누르면 **바로 이동**한다(확인 버튼 없음). 지금 옮기는 폴더(와 그 안)는 목적지 목록에서 빠진다.
- **삭제:** 예전과 같이 확인을 거쳐 지운다. **폴더를 고르면 안의 항목까지** 함께 지운다.
- **모바일 전용**(데스크탑 북마크 목록엔 이 진입점이 원래 없으므로 변화 없음).

### 함께 고친 작은 버그 — 삭제 확인을 취소하면 줄이 닫힌다

행을 옆으로 밀어(스와이프) 삭제를 누른 뒤 **확인 창에서 "취소"를 누르면, 예전엔 그 줄이 삭제 버튼이 보이는 채로 열려 있었다**(수정은 정상이었는데 삭제만 어긋났다). 이제 삭제도 누르는 즉시 줄을 닫아, 확인을 취소하면 줄이 제자리로 돌아온다.

### 링크 주소는 한 곳에서 바꾼다

공유 링크의 도메인은 `SITE_BASE`라는 **단일 상수**로 뽑아 뒀다. 나중에 주소가 바뀌어도(예: 기도서까지 합친 `bok.anglican.kr`) 이 한 곳만 고치면 모든 공유 링크가 따라온다.

## 무엇이 더 안전/깔끔해졌는가

같은 항목 목록을 별도 창에 **한 벌 더 그리던 코드·스타일을 통째로 들어냈다**(서로 어긋날 여지 감소). 삭제 확인 취소 시 줄이 어정쩡하게 열려 있던 혼란도 없앴다.

새로 추가·정리한 자동 점검(`bookmark.js` 단위 **149건**, 전체 회귀 **660건** 통과, 타입 검사 0건 경고):

### 선택 셈 — 7건
어느 폴더에 속하는지 추적 / 폴더를 고르면 그 안 전부가 "대상"으로 잡힘(깊이 무관) / 제거될 개수를 빠짐없이·겹치지 않게 셈 / 실제 삭제 대상은 "가장 바깥 선택"만.

### 공유 링크 묶음 — 3건 (`_collectSelectedBookmarks`)
폴더를 고르면 그 안의 **북마크 잎**만 펼쳐 모음(폴더 자체는 링크가 없으므로) / 직접 고른 북마크도 포함 / 아무것도 안 고르면 빈 목록.

### 공유 payload — 2건 (`_buildSharePayload`)
1개 → `{title, url}`(절대 링크) / 여러 개 → `{title, text}`(제목+링크 줄 목록). 둘 다 `SITE_BASE` 절대 주소.

### 아래 안내 문구 — 2건
아무것도 안 골랐을 때 "항목을 선택하세요", 고르면 "N개 선택됨".

## 자동 점검이 닿지 않는 부분 (수동/e2e 확인)

선택 표시 슬라이드인, 아래 막대가 탭 바를 대체하는 전환, 메뉴 "선택" 진입, 공유 시트 호출(헤드리스에선 `navigator.share` 를 가짜로 끼워 payload 검증), 이동 모달에서 폴더 탭 시 실제 이동, 삭제 전체 흐름은 브라우저 자동 테스트로 확인: `tests/e2e/test_bookmark_select_delete.py` 9건(진입·선택·폴더 cascade·전체 선택·공유·이동·취소·확인취소·빈 목록), 폴더 삭제(내용까지) `test_bookmark_folders.py::test_folder_delete`, 삭제 확인 취소 시 줄 닫힘 `test_bookmark_swipe.py::test_cancel_delete_confirm_closes_swipe`. 모바일 화면에서 눈으로도 확인(3-액션 알약·이동 모달·메뉴 순서).
